Maven 入门

编写 POM

POM (Project Object Model, 项目对象模型) 定义了项目的基本信息, 用于描述项目如何构建, 声明项目依赖, 等等

<!-- project是所有pom.xml的根元素, 它还声明了一些POM相关的命名空间及xsd元素 -->
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <!-- 根元素下的第一个子元素modelVersion指定了当前POM模型的版本 -->
    <modelVersion>4.0.0</modelVersion>
    <!-- groupId定义了项目属于哪个组, 这个组往往和项目所在的组织或公司存在关联 -->
    <groupId>com.janz.maven</groupId>
    <!-- artifactId定义了当前Maven项目在组中唯一的ID -->
    <artifactId>hello-world</artifactId>
    <!-- version指定了Hello World项目当前的版本 -->
    <version>1.0-SNAPSHOT</version>
    <packaging>jar</packaging>
    
    <!-- name元素声明了一个对于用户更为友好的项目名称 -->
    <name>hello-world</name>
    <url>http://maven.apache.org</url>

    <properties>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <!-- 添加了dependencies元素, 该元素下可以包含多个dependency元素以声明项目的依赖 -->
    <dependencies>
        <!-- 添加JUnit依赖 -->
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <!-- scope为依赖范围, 默认值compile, 表示该依赖对主代码和测试代码都有效, 若依赖范围为test则表示该依赖只对测试有效 -->
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>

没有任何实际的 Java 代码, 就能够定义一个 Maven 项目的 POM, 这体现了 Maven 的一大优点, 它能让项目对象模型最大程度地与实际代码相独立, 可以称之为解耦, 或者正交性, 这在很大程度上避免了 Java 代码和 POM 代码的相互影响, 比如当项目需要升级版本时, 只需要修改 POM, 而不需要更改 Java 代码, 而在 POM 稳定之后, 日常的 Java 代码开发工作基本不涉及 POM 的修改

编写主代码

默认情况下, Maven 假设项目主代码位于 src/main/java 目录

package com.janz.maven;

public class HelloWorld {

	public String sayHello() {
		return "Hello Maven";
	}
    
    public static void main(String[] args) {
		System.out.println(new HelloWorld().sayHello());
	}
}

关于该 Java 代码有两点需要注意:

  • 在绝大多数情况下, 应该把项目主代码放到 src/main/java/ 目录下 (遵循 Maven 的约定), 而无须额外的配置, Maven 会自动搜寻该目录找到项目主代码;
  • 该 Java 类的包名是 com.janz.maven, 这与之前在 POM 中定义的 groupId 和 artifactId 相吻合, 一般来说, 项目中 Java 类的包都应该基于项目的 groupId 和 artifactId, 这样更加清晰, 更加符合逻辑, 也方便搜索构件或者 Java 类

旧版本的 Maven 的核心插件 compiler 默认只支持编译 Java 1.3, 因此需要配置该插件使其支持 Java 5

<!-- 旧版本需要配置maven-compiler-plugin支持Java 5 -->
<project>
    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
                    <source>1.5</source>
                    <target>1.5</target>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>

编写测试代码

项目主代码和测试代码不同, 项目的主代码会被打包到最终的构件中 (如 jar), 而测试代码只在运行测试时用到, 不会被打包

为了使项目结构保持清晰, 主代码与测试代码应该分别位于独立的目录中, Maven 项目中默认的测试代码目录是 src/test/java

package com.janz.maven;

import org.junit.Assert;
import org.junit.Test;

public class HelloWorldTest {

	@Test
	public void testSayHello() {
        // 准备测试类及数据
		HelloWorld helloWorld = new HelloWorld();
        // 执行要测试的行为
		String result = helloWorld.sayHello();
        // 检查结果
		Assert.assertEquals(result, "Hello Maven");
	}
}

常用命令

λ mvn clean
# 清理项目, 删除target文件夹

λ mvn compile
# 编译项目, 不会编译测试项目

λ mvn test-compile
# 编译测试项目, 会先编译主项目

λ mvn test
# 运行测试项目

λ mvn site
# 生成报告, 需配置

λ mvn package
# 打包项目, 默认jar, 不包含源码
# jar:jar任务负责打包, 实际上就是jar插件的jar目标将项目主代码打包成一个名为hello-world-1.0-SNAPSHOT.jar的文件, 该文件也位于target/输出目录中, 它是根据artifact-version.jar规则进行命名的, 如有需要, 还可以使用finalName来自定义该文件的名称

λ mvn source:jar
λ mvn source:jar-no-fork
# 打包源码, 会根据artifact-version-sources.jar规则生成源码jar包
# 优先使用jar-no-fork

λ mvn install
# 安装到本地仓库

λ mvn deploy
# 发布到远程私服

λ mvn archetype:generate
λ mvn archetype:generate -DarchetypeCatalog=internal
# 运行生成项目模板, -DarchetypeCatalog=internal指定只使用archetype内置的catalog

其他命令

λ mvn eclipse:eclipse
# 生成eclipse项目的配置文件

λ mvn eclipse:clean
# 清除eclipse项目的配置文件

λ mvn cargo:run
λ mvn cargo:start
# 运行cargo指定的容器

λ mvn cargo:deploy
# 将项目war包发布到cargo指定的容器

运行

默认打包生成的 jar 是不能够直接运行的, 因为带有 main 方法的类信息不会添加到 manifest 中 (打开 jar 文件中的 META-INF/MANIFEST.MF 文件, 将无法看到 Main-Class 一行)

<!-- 为了生成可执行的jar文件, 需要借助maven-shade-plugin, 配置该插件如下 -->
<project>
<build><plugins><plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-shade-plugin</artifactId>
    <version>1.2.1</version>
    <executions>
        <execution>
            <phase>package</phase>
            <goals>
                <goal>shade</goal>
            </goals>
            <configuration>
                <transformers>
                    <transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
                        <!-- 配置mainClass, 项目在打包时会将该信息放到MANIFEST中 -->
                        <mainClass>com.janz.maven.HelloWorld</mainClass>
                    </transformer>
                </transformers>
            </configuration>
        </execution>
    </executions>
    </plugin></plugins></build>
</project>
# 重新打包项目
λ mvn clean package

# 构建完成之后打开target/目录, 可以看到hello-world-1.0-SNAPSHOT.jar和original-hello-world-1.0-SNAPSHOT.jar, 前者是带有Main-Class信息的可运行jar, 后者是原始的jar
# 在项目根目录中执行jar文件
λ java -jar target\hello-world-1.0-SNAPSHOT.jar
