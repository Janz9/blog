JPA 扩展查询

QBE(QueryByExample)

按 实例查询(QBE) 是一种用户友好的查询技术, 具有简单的接口, 它允许动态查询创建, 并且不需要编写包含字段名称的查询, 继承 QueryByExampleExecutor 和 JpaRepository 都会有这些基本方法

/* QueryByExampleExecutor源码 */
public interface QueryByExampleExecutor<T> {
    <S extends T> S findOne(Example<S> example);
    <S extends T> Iterable<S> findAll(Example<S> example);
    <S extends T> Iterable<S> findAll(Example<S> example, Sort sort);
    <S extends T> Page<S> findAll(Example<S> example, Pageable pageable);
    <S extends T> long count(Example<S> example);
    <S extends T> boolean exists(Example<S> example);
}

使用示例

/* 创建查询条件 */
@Service
public class UserService {

    @Autowired
    private UserRepository userRepositoryImpl;
    
    public Optional<UserDO> findByName(String name) {
        UserDO user = new UserDO();
        user.setId(10L);
        user.setName(name);
        // 创建匹配器, 即如何使用查询条件
        ExampleMatcher matcher = ExampleMatcher.matching()
            .withIgnoreCase(true)
            .withMatcher("name", ExampleMatcher.GenericPropertyMatchers.contains())
            .withIgnorePaths("id");
        // 创建实例
        Example<UserDO> example = Example.of(user, matcher);
        // 查询
        return userRepositoryImpl.findOne(example);
    }
}

特点 / 约束

实例查询顾名思义, 就是通过一个例子来查询:

  • 支持动态查询 即支持查询条件个数不固定的情况;
  • 不支持过滤条件分组 即不支持过滤条件用 or 来连接, 所有的过滤条件都是简单用 and 连接;
  • 仅支持一个过滤条件值 要查询的条件值只有一个存储过滤值的位置, 没办法同时传入两个过滤值

ExampleMatcher 配置

nullHandler 标识作为条件的实体对象中, 一个属性值为 null 时, 表示是否参与过滤

  • INCLUDE 表示参与过滤, 会匹配数据库表中该字段值是 NULL 的记录;
  • IGNORE 表示不参与过滤

defaultStringMatcher 默认字符串匹配方式

  • DEFAULT 默认值, 效果同 EXACT;
  • EXACT 相等;
  • STARTING 开始匹配;
  • ENDING 结束匹配;
  • CONTAINING 包含匹配;
  • REGEX 正则表达式

defaultIgnoreCase 默认大小写忽略方式, 布尔型

propertySpecifiers 各属性特定查询方式, 描述了各个属性单独定义的查询方式

ignoredPaths 忽略属性列表, 忽略的属性不参与查询过滤

QBC(QueryByCriteria)

Root<T> 代表了可以查询和操作的实体对象的根, 通过对象的 Path<Y> get(String attributeName) 来获得操作的字段

CriteriaQuery<?> 代表了一个顶层查询对象, 包含着查询的各个部分, 比如 select, from, where, group by, order by 等

CriteriaBuilder 用来构建 CritiaQuery 的构建器对象, 相当于条件或者是条件组合, 以谓语即 Predicate 的形式返回

使用示例

/* 动态查询 */
public List<UserDO> findByCriteria(String name, String phone, String email) {
    CriteriaBuilder cb = entityManager.getCriteriaBuilder();
    CriteriaQuery<UserDO> query = cb.createQuery(UserDO.class);
    Root<UserDO> root = query.from(UserDO.class);

    if (!StringUtils.isEmpty(name)) {
        query.where(cb.like(root.get("name"), "%" + name + "%"));
    }
    if (!StringUtils.isEmpty(phone)) {
        query.where(cb.equal(root.get("phone"), phone));
    }
    if (!StringUtils.isEmpty(email)) {
        query.where(cb.equal(root.get("email"), email));
    }
    return entityManager.createQuery(query).getResultList();
}

JSE(JpaSpecificationExecutor)

Spring Data JPA 支持 Criteria 查询, 可以很方便的使用, JpaSpecificationExecutor 是 JPA 2.0 提供的 Criteria API, 可以用于动态生成 query

/* JpaSpecificationExecutor源码 */
public interface JpaSpecificationExecutor<T> {
    Optional<T> findOne(Specification<T> spec);
    List<T> findAll(Specification<T> spec);
    Page<T> findAll(Specification<T> spec, Pageable pageable);
    List<T> findAll(Specification<T> spec, Sort sort);
    long count(Specification<T> spec);
}
/* Specification源码 */
public interface Specification<T> extends Serializable {
    static <T> Specification<T> not(Specification<T> spec) {
		return Specifications.negated(spec);
	}
	static <T> Specification<T> where(Specification<T> spec) {
		return Specifications.where(spec);
	}
	default Specification<T> and(Specification<T> other) {
		return Specifications.composed(this, other, AND);
	}
	default Specification<T> or(Specification<T> other) {
		return Specifications.composed(this, other, OR);
	}
	Predicate toPredicate(Root<T> root, CriteriaQuery<?> query, CriteriaBuilder criteriaBuilder);
}

使用示例

/* 动态查询 */
public Page<UserDO> findBySpecification(String name, String phone, String email, Pageable pageable) {
    return userRepositoryImpl.findAll((root, query, cb) -> {
        List<Predicate> predicates = new ArrayList<>();
        if (!StringUtils.isEmpty(name)) {
            predicates.add(cb.like(root.get("name"), "%" + name + "%"));
        }
        if (!StringUtils.isEmpty(phone)) {
            predicates.add(cb.equal(root.get("phone"), phone));
        }
        if (!StringUtils.isEmpty(email)) {
            predicates.add(cb.equal(root.get("email"), email));
        }
        return query.where(predicates.toArray(new Predicate[predicates.size()])).getRestriction();
    }, pageable);
}

自定义 Repository

由于业务场景的差别, 有可能需要定义自己的 Repository 类, 有两种方式:

  • 继承 JpaSpecificationExecutor 通过 CriteriaQuery 几乎可以实现任何逻辑;
  • 写独立接口继承 CrudRepository, 独立实现类利用 EntityManager 操作 Criteria Query API 可以实现任何逻辑

EntityManager

/* 使用EntityManager查询 */
public class UserRepositoryImpl {

    @PersistenceContext
    private EntityManager entityManager;

    public Optional<UserDO> findById(Long id) {
        UserDO user = null;

        // 使用原生SQL查询
        String sql = "select * from b_user u where u.id = ?1";
        user = (UserDO) entityManager.createNativeQuery(sql).setParameter(1, id).getSingleResult();
        // 使用JPQL查询
        String jpql = "select u from UserDO u where u.id = ?1";
        user = (UserDO) entityManager.createQuery(jpql).setParameter(1, id).getSingleResult();
        // 使用find方法
        user = entityManager.find(UserDO.class, id);
        
        return Optional.of(user);
    }
}
