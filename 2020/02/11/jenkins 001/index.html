Jenkins 简介

简介

Jenkins

Jenkins 是一款使用 Java 语言开发的开源的自动化服务器, 通过界面或 Jenkinsfile 告诉它执行什么任务, 何时执行, 理论上, 可以让它执行任何任务, 但是通常只应用于持续集成和持续交付

Jenkinsfile 是一个文本文件, 也就是部署流水线概念在 Jenkins 中的表现形式, 所有部署流水线的逻辑都写在 Jenkinsfile 中

安装

Jenkins 官网提供了 war 文件, 安装程序和 Docker 镜像的下载链接

镜像

配置国内加速镜像 参考

Pipeline

部署流水线(Deployment pipeline) 是指从软件版本控制库到用户手中这一过程的自动化表现形式

Jenkins 1.x 只能通过界面手动操作来描述部署流水线, Jenkins 2.x 支持 pipeline as code, 可以通过代码来描述部署流水线

pipeline 语法

  • 脚本式(Scripted)

    Groovy 语言被选择作为基础来实现 pipeline, 所以在写脚本式 pipeline, 其实就是在写 Groovy 代码, 这样为用户提供了巨大的灵活性和可扩展性

  • 声明式(Declarative)

    Groovy 语言的学习成本对于不使用 Groovy 的开发团队来说通常是不必要的, 所以 Jenkins 提供了更简单, 更结构化的声明式语法

Pipeline 项目

要创建 pipeline, 首先要安装 pipeline 插件, 在 Jenkins 中新建一个 pipeline 项目

pipeline script

在 pipeline 节点下选择 Pipeline script, 脚本中填入示例代码

直接在 Jenkins 界面上填入 pipeline 脚本内容, 在试验时可以这么做, 但是不推荐, 因为这样无法做到 pipeline 的版本化

pipeline script from SCM

从代码仓库拉取 pipeline 并执行, 在 pipeline 节点下选择 Pipeline script from SCM, 并在 SCM 中选择对应的仓库类型, 项目结构新建 Jenkinsfile 文件, 示例代码和执行结果同上

Multibranch Pipeline 项目

实际项目中, 往往需要多分支同时进行开发, 如果为每个分支都分别创建 Jenkins 项目有些多余, Jenkins 支持多分支 pipeline, 创建此类项目时, 需要选择 Multibranch Pipeline

分支扫描根据一定的策略去代码仓库扫描分支, 如果有新分支就创建一个以分支命名的任务, 如果分支被删除了, 就删除相应的任务

Maven 构建

Maven 是非常流行的一个 Java 应用构建工具, Jenkins 默认支持 Maven

首先需要在 Jenkins 上安装 JDK 和 Maven, 可以登录 Jenkins 服务器手动安装, 也可以让 Jenkins 自动安装

进入 Manage Jenkins / Global Tool Configuration / Maven 页配置自动安装的 Maven
