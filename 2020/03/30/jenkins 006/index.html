扩展插件

扩展

在大量使用 pipeline 后会发现 Jenkins 内置的功能并不能照顾到所有的需求, 这时就需要扩展 pipeline

函数

pipeline 本质是一个 Groovy 脚本, 所以可以在 pipeline 中定义函数, 并使用 Groovy 语言自带的特性

/* 示例: 定义函数 */
pipeline {
    agent any
    stages {
        stage('build') {
            steps {
                echo "${createVersion(BUILD_NUMBER)}"
            }
        }
    }
}
def createVersion(String BUILD_NUMBER) {
    return new Date().format('yyMM') + "-${BUILD_NUMBER}"
}

共享库

Jenkins pipeline 提供了 共享库(Shared library) 技术, 可以将重复代码定义在一个独立的代码控制仓库中, 其他的 Jenkins pipeline 加载使用它

创建共享库项目并推送到 SCM, 在 Manager Jenkins / Configure System / Global Pipeline Libraries 节点下配置全局共享库

/* 示例: 共享库结构 */
├─src
│  └─codes
│     └─showme
│        └─Utils.groovy
└─vars
   └─sayHello.groovy

// src目录是一个标准的Java源码结构, 目录中的类被称为库类
// vars目录是可以在pipeline中直接调用的全局变量, 变量的文件名即调用的函数名
/* 文件: src/codes/showme/Utils.groovy */
package codes.showme
class Utils implements Serializable {
    def getVersion(String BUILD_NUMBER, String GIT_COMMIT) {
        return new Date().format('yyMM') + "-${BUILD_NUMBER}" + "-${GIT_COMMIT}"
    }
}
// 库类可以使用@Grab注解使用第三方依赖
/* 文件: vars/sayHello.groovy */
def call(String name) {
    echo "hello ${name}."
}
// 全局变量实际调用的是文件中的call函数, 函数中可以直接调用pipeline的步骤, 函数参数还支持接收闭包
/* 示例: 共享库 */
@Library('global-shared-library') // 使用@Library指定共享库, 需要与全局共享库配置的Name一致
pipeline {
    agent any
    stages {
        stage('build') {
            steps {
                sayHello('world') // 使用共享库vars目录中定义的全局变量
                script { // 使用src目录中的类时, 需要使用全包名, 同时因为写的是Groovy代码, 所以需要使用script指令
                    def util = new codes.showme.Utils()
                    def v = util.getVersion("${BUILD_NUMBER}", "${GIT_COMMIT}")
                    echo "${v}"
                }
            }
        }
    }
}
// @Library语法
@Library('Name@version')
// Jenkins支持添加多个共享库, 所以@Library注解允许同时引入多个共享库, 出现同名函数时的处理方式是先定义者生效
@Library(['Name@version', 'Name2@version2'])

共享库模板

声明式 pipeline 在 1.2 版本后, 可以在共享库中定义 pipeline, 通过此特性, 可以定义 pipeline 的模板, 如果大多数项目的结构都是标准化的, 那么利用 pipeline 模板技术可以大大降低维护 pipeline 的成本

/* 示例: 共享库代码 */
/* vars/generatePipeline.groovy */
def call(String lang) {
    if(lang == 'go') {
        pipeline {
            // 代码同pipeline定义
        }
    } else if(lang == 'java') {
        pipeline {
            // 代码同pipeline定义
        }
    }
}
/* 示例: 使用模板 */
@Library('global-shared-library')
generatePipeline('go')

插件

Jenkins 插件开发文档 TUTORIAL, 使用 Maven 的 archetype 插件生成代码骨架, 非常简单

# 示例: 使用mvn命令生成项目
mvn archetype:generate -Dfilter=io.jenkins.archetypes:hello-world-plugin

# 生成的代码骨架
├───pom.xml
└───src
    ├───main
    │   ├───java
    │   │   └───io
    │   │       └───jenkins
    │   │           └───plugins
    │   │               └───sample
    │   │                   └───HelloWorldBuilder.java
    │   └───resources
    │       ├───index.jelly
    │       └───(省略i18n文件)
    └───test
        └───java
            └───io
                └───jenkins
                    └───plugins
                        └───sample
                            └───HelloWorldBuilderTest.java

# pom.xml的<name>标签     定义插件的名称
# HelloWorldBuilder.java  步骤具体实现类
# index.jelly             插件管理页面显示的HTML内容
/* 文件: HelloWorldBuilder.java */
public class HelloWorldBuilder extends Builder implements SimpleBuildStep {

    private final String name;
    private boolean useFrench;

    // @DataBoundConstructor注解标识插件类的构造函数, name属性为插件默认属性, 也是调用插件时的必要参数
    @DataBoundConstructor
    public HelloWorldBuilder(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public boolean isUseFrench() {
        return useFrench;
    }

    // @DataBoundSetter注解标识插件属性的setter方法
    @DataBoundSetter
    public void setUseFrench(boolean useFrench) {
        this.useFrench = useFrench;
    }

    @Override
    public void perform(Run<?, ?> run, FilePath workspace, Launcher launcher, TaskListener listener) throws InterruptedException, IOException {
        // 插件的执行逻辑
        if (useFrench) {
            listener.getLogger().println("Bonjour, " + name + "!");
        } else {
            listener.getLogger().println("Hello, " + name + "!");
        }
    }

    // 扩展点唯一标识, 被定义在扩展点上, 可以理解为在pipeline中使用插件时所引用的函数名, 本例的扩展点标识为greet
    @Symbol("greet")
    // 扩展点注解, 通过此注解自动发现扩展点, 并将扩展点加入扩展点列表中
    @Extension
    public static final class DescriptorImpl extends BuildStepDescriptor<Builder> {

        public FormValidation doCheckName(@QueryParameter String value, @QueryParameter boolean useFrench)
                throws IOException, ServletException {
            if (value.length() == 0)
                return FormValidation.error(Messages.HelloWorldBuilder_DescriptorImpl_errors_missingName());
            if (value.length() < 4)
                return FormValidation.warning(Messages.HelloWorldBuilder_DescriptorImpl_warnings_tooShort());
            if (!useFrench && value.matches(".*[éáàç].*")) {
                return FormValidation.warning(Messages.HelloWorldBuilder_DescriptorImpl_warnings_reallyFrench());
            }
            return FormValidation.ok();
        }

        @Override
        public boolean isApplicable(Class<? extends AbstractProject> aClass) {
            return true;
        }

        @Override
        public String getDisplayName() {
            return Messages.HelloWorldBuilder_DescriptorImpl_DisplayName();
        }
    }
}

Jenkins 插件开发的代码骨架中, pom.xml 文件默认添加了用于插件开发的 Maven 插件 maven-hpi-plugin, 通过该插件, 只需要实现插件代码, 然后执行 mvn hpi:run 就可以启动一个安装了插件的 Jenkins 实例, 在浏览器中打开 http://localhost:8080/jenkins 即可访问

/* 示例: 在Jenkinsfile中使用插件 */
pipeline {
    agent any
    stages {
        stage('build') {
            steps {
                greet(name: 'build', useFrench: false)
            }
        }
    }
}

// greet是通过@Symbol定义的插件扩展点名称, 可以将这个名称理解为一个函数名, @DataBoundConstructor和@DataBoundSetter注解的方法可以理解为这个函数的参数
