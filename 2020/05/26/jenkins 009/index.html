Jenkins 管理

发表于 2020-05-26 更新于 2021-05-06 分类于 运维 阅读次数：
本文字数： 5.8k 阅读时长 ≈ 5 分钟

管理

制品

制品 是软件开发过程中产生的多种有形副产品之一, 狭义的制品可以简单地理解为二进制包, 广义的制品还包括用例, UML 图, 设计文档等

最简单的制品管理仓库就是将制品统一放在一个系统目录结构下, 但是更多的做法是使用现成的制品库, 目前现成的制品库有 Nexus, Artifactory 等

Maven 制品

Maven 团队提供的官方插件支持将打包后的制品发布到配置的仓库中

Nexus 提供了 Nexus Platform 插件发布 Maven 制品, 安装好插件后, 在 Manage Jenkins / Configure System / Sonatype Nexus 节点下配置 Nexus 服务器地址

Pipeline 制品

可以使用 archiveArtifacts 指令对制品进行归档, 构建的结果页面提供了归档文件的下载链接

在某些场景下, 需要在 pipeline 之间拷贝制品, 使用 Copy Artifact 插件可以实现

插件会将拷贝的制品的原构建序号放到环境变量 COPYARTIFACT_BUILD_NUMBER_(resultVariableSuffix) 中, 在当前 pipeline 中可以通过此变量拿到原构建序号

版本号

版本号 的制定并没有所谓的行业标准, GitHub 提出了一种具有指导意义, 统一的版本号表示规则, 称为 Semantic Versioning(语义化版本表示), 先行版本号及版本编译元数据可以加到后面作为延伸

Jenkins 的插件 Version Number 是一款用于生成版本号的插件, 提供了 VersionNumber 指令

通知

实时知道构建的状态很重要, 最好的方式就是构建系统本身知道出现情况时发起通知

邮件通知

邮件通知 是最常用的通知方式, Jenkins 默认支持

进入 Manage Jenkins / Configure System / Jenkins Location 页面, 设置管理员邮箱, 并配置 Email Notification 节点的信息

Email Extension 插件对邮件通知的功能进行了扩展, 支持更多的特性

进入 Manage Jenkins / Configure System / Extended Email Notification 页面配置信息

HTTP 请求通知

HTTP Request 插件能在 Jenkins pipeline 中发送 HTTP 请求给第三方系统

视图

Green Balls 插件的作用让构建成功的状态图标变成绿色的

Build Monitor View 插件可以将 Jenkins 项目以一块看板的形式呈现

监控

Prometheus 是一款开源的监控和告警系统, 服务器端主动从客户端拉取指标数据, Jenkins 的 Prometheus 插件的目的就是为了暴露一个数据接口, 这样 Prometheus 就可以拉取到指标数据了

Prometheus 本身提供的界面过于简陋, 所以一般会使用 Grafana 对指标进行展示

graph LR
A(Jenkins master)
B(Prometheus)
C(Grafana)
B --pull--> A
C --query--> B

备份

Jenkins 的所有数据都是存放在文件中的, 所以 Jenkins 备份其实就是备份 JENKINS_HOME 目录

如果启动 Jenkins 时没有指定 JENKINS_HOME 环境变量, 那么 Jenkins 将在当前用户目录下创建一个 .jenkins 目录作为 JENKINS_HOME 目录

Jenkins 本身并不提供备份功能, 而是交给插件来完成, 安装 Periodic Backup 插件后, 在 Manage Jenkins / Periodic Backup Manager 节点下管理备份功能
