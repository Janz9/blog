cURL 进阶

发表于 2020-05-19 更新于 2021-05-31 分类于 工具 阅读次数： 本文字数： 4.6k 阅读时长 ≈ 4 分钟

进阶

curl 很少会自己决定做什么, 它试图在最大程度上让开发人员做自己想做的事, 这是一种设计选择, 它允许通过传递选项的方式, 调整 curl 与服务器的交互

调试

当执行 curl 命令没有返回所期望的数据时, 可以通过选项来输出更多详细的信息

详细模式

使用 -v 或 --verbose 选项来启用 详细模式

跟踪信息

当想要保存包括实际传输数据在内的完整信息时, curl 提供了两个选项

打印信息

curl 提供了 --write-out 选项在传输任务完成后打印一些信息, 并且它提供了大量可添加到输出内容中的变量, 这些变量包含了与传输相关的信息

选项的可用变量见文档 NOTE

静默模式

与详细模式相反的是 静默模式, 可以使用 -s 或 --silent 选项让 curl 关闭进度指示器, 而且不会在发生错误时输出任何错误信息

启用静默模式后, 可以通过使用 -S 或 --show-error 来要求它在发生错误时输出错误信息

持久连接

在建立 TCP 连接时, curl 将保留旧连接一段时间, 如果下一次要连接到同一主机, 那么就可以重用相同的连接, 从而节省大量时间, 称之为 持久连接, 不过 curl 命令行工具只能在运行期间保持连接处于活跃状态, 因此只要它退出, 就会关闭所有已打开的连接, 这种活跃连接池称为 连接缓存

如果需要针对同一主机或相同的 URL 执行 N 次传输或操作, 那么可以考虑使用尽可能少的 curl 命令行来提速, 而不是每次使用一个 URL 并重复调用 curl

存储指令

curl 默认下载 URL 指定的资源, 将输出内容打印到终端上

HTTP 服务器可以选择在响应消息中提供 Content-Disposition 标头, 这个标头可能包含服务器建议的文件名

存储指令的读取和处理顺序与下载的 URL 的顺序相同, 因此以下写法是等效的

间隔符

curl 提供了 -: 或 --next 选项, 用于在一组选项和 URL 之间插入间隔, 当命令行解析器遇到 --next 选项时, 它会将后面的选项应用于下一组 URL

退出码

curl 会在处理完最后一个 URL 后返回一个退出码, 一切正常则返回 0, 如果想让 curl 在第一次出现错误时就退出, 可以使用 --fail-early 选项

可用的退出码见文档 EXIT CODES

以非零状态码退出时, curl 还会输出一个错误消息, 这个错误消息可能会包含一些额外的信息, 因此, 相同的错误码可能会与不同的错误消息一起出现

连接

在 curl 中使用的大多数协议都是基于 TCP 的, 连接意味着需要进行 TCP 协议握手

主机

通常同一个 HTTP 服务器实例会有多个名字, 自定义的 Host: 标头可以让服务器返回目标站点的内容, 即使没有使用目标站点的主机名进行连接

可以通过 --resolve 选项将 IP 地址插入 DNS 缓存中, 以便解析得到自己想要的地址

可以通过 --connect-to 选项为 curl 指定特定主机名和端口

代理

代理是代表客户端执行某项操作的机器或软件, curl 支持几种不同类型的代理, curl 使用 -x 或 --proxy 选项指定代理, 默认代理类型是 HTTP, 端口 3128

可以通过设置 [SCHEME]_PROXY 变量来指定代理, curl 会在运行前检查这些特殊的环境变量, 然后根据这些变量决定是否使用代理

可以通过单个变量 ALL_PROXY 来设置所有的协议, 如果存在某个特定的协议变量, 则优先使用这个变量

使用环境变量设置代理时, 可以使用 NO_PROXY 变量将无须使用代理的主机名排除在外, 使用逗号分隔多个主机名, 也可以使用单个星号 * 以匹配所有主机

超时

curl 的大多数操作默认没有设置超时

可以用 -m 或 --max-time 选项设置 curl 最长有多少时间可用, 以秒为单位

可以用 --connect-timeout 限制 curl 连接主机所花费的时间, 以秒为单位

作为固定超时时间的替代方案, 可以设置如果传输速率低于某个特定值, 并且在某个时间段内一直低于这个值, 那么就放弃传输

curl 默认启用 TCP keepalive, 可以使用 --keepalive-time 来指定发送探测的频率, 默认为 60 秒
