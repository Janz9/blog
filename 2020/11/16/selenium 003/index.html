Selenium 设计模式

发表于 2020-11-16 更新于 2021-05-25 分类于 测试 阅读次数：

设计模式

BDD

BDD(Behaviour-Driven Development) 行为驱动开发是一种敏捷的软件开发技术, 使用自然语言编写用例的好处是能够让非技术人员 (客户和用户) 看懂测试用例, 可以最大程度的减少技术语言和领域语言的翻译成本

Cucumber

Cucumber 是一款支持行为驱动开发, 读取以纯文本形式编写的可执行规范, 验证该软件是否满足规范的内容, 并生成报告, 官网 Cucumber

Gherkin

Cucumber 运行时会从纯文本语言编写的 .feature 文件中读取测试规格说明, 解析需要测试的场景 scenario, 每个场景由一系列步骤 step 组成, 为了使 Cucumber 理解特性文件, 这些文件必须遵循 Gherkin 的基本语法规则

Gherkin 使用一组特殊关键字为可执行规范指定结构和含义, 为了让 Gherkin 能用多种语言书写, 关键字已经被翻译成多种语言, 为了提高可读性和流畅性, 有些语言可能对任何给定的关键字有多个翻译, 列表 languages

# 示例: login.feature
Feature: 登录测试
  使用手机号码和密码登录并跳转团队首页

  Scenario:
    Given 访问登录页面
    And 输入手机号码 "135xxxxxxxx"
    And 输入密码 "xxxxxxxx"
    When 点击登录按钮
    Then 跳转团队首页

可通过文件头添加 # language: 确定要使用什么样的语言, 如果省略此属性将默认使用英语 en

# language: zh-CN

Keyword

Cucumber 关键字查看 keywords

Feature 每个文件都以 Feature 关键字开头, 后跟特性的名称和描述, 描述可以跨越多行, 该关键字完全不影响测试行为, 但是会包含在生成的报告中

Rule 表示一个被实现的业务规则, 用于将属于此业务规则的多个场景组合在一起

Example / Scenario 具体测试场景, 它由步骤列表组成, Scenario 关键字后面也可以跟上名称和描述

Step 在 Scenario 中定义的步骤列表, 一般使用 Given, When, Then, But 和 And 这些关键词, Step 实际上并不关心使用哪个关键字, 关键字的选择只是为了创建可读性更好的场景

Given 执行用例的前置条件

When 执行用例的交互方式

Then 断言, 检查交互的结果是否符合预期

And 同前一个步骤

But 反前一个步骤

Gherkin 支持使用星号 * 代替任何常规 step 关键字

Background 在所有场景中都会重复的步骤, 可以通过将这些步骤分组到 Backgroud, 背景放在第一个场景之前

Scenario Outline / Scenario Template 可以使用不同的参数组合多次测试同一场景

示例

示例基于 Cucumber 和 Junit5 测试环境

<!-- 配置: pom.xml添加依赖 -->
<dependency>
    <groupId>io.cucumber</groupId>
    <artifactId>cucumber-java</artifactId>
    <version>${cucumber.version}</version>
</dependency>
<dependency>
    <groupId>io.cucumber</groupId>
    <artifactId>cucumber-junit</artifactId>
    <version>${cucumber.version}</version>
    <scope>test</scope>
</dependency>

将文件 login.feature 拷贝到项目文件夹 /src/test/resources/feature 下, 编写实现类和测试类

/* 示例: feature定义实现类 */
public class Login {
    private WebDriver driver;
    @Before
    public void start() {
        System.setProperty("webdriver.edge.driver", "driver/msedgedriver.exe");
        driver = new EdgeDriver();
        driver.manage().window().maximize();
        driver.manage().timeouts().pageLoadTimeout(10L, TimeUnit.SECONDS);
    }
    @After
    public void end() {
        driver.quit();
    }
    @Given("访问登录页面")
    public void loginPage() {
        driver.get("http://news.anhuai.com/user/login");
    }
    @And("输入手机号码 {string}")
    public void inputPhone(String phone) {
        driver.findElement(By.id("username")).sendKeys(phone);
    }
    @And("输入密码 {string}")
    public void inputPassword(String pwd) {
        driver.findElement(By.id("password")).sendKeys(pwd);
    }
    @When("点击登录按钮")
    public void submit() {
        driver.findElement(By.className("ant-btn")).click();
    }
    @Then("跳转团队首页")
    public void success() {
        Thread.sleep(2000L);
        String url = driver.getCurrentUrl();
        Assertions.assertTrue(StringUtils.startsWith(url, "http://news.anhuai.com/team/"));
    }
}
/* 示例: 测试类 */
@RunWith(Cucumber.class)
@CucumberOptions(
    // 指定feature文件夹的位置
    features = {"classpath:/feature"},
    // 指定插件的配置
    plugin = {"pretty", "html:target/cucumber.html", "json:target/cucumber.json"}, 
    // 指定实现类的包路径
    glue = {"com.anhuai.test.demo"} 
)
public class CucumberTest { }

TDD

TDD(Test-Driven Development) 是敏捷开发中的一项核心实践和技术, 原理是在开发功能的代码前, 先编写单元测试用例代码, 测试代码确定需要编写的具体产品代码

数据驱动 设计是自动化测试中非常重要的一个环节, 同时也是自动化测试中的主流设计模式之一, 主要包括使用 DataProvider, CSV, Excel, YAML 和 MySQL 等方法进行数据驱动测试

在实际设计测试用例的过程中, 经常会用等价类或边界值这样的方法, 针对一个功能用例测试数据进行测试, 比如一个输入框的正向数据, 逆向数据, 非法输入等, 即输入不同类型的数据进行测试

示例

# 示例: login.feature
Feature: 登录测试
  使用手机号码和密码登录并跳转团队首页

  Scenario:
    Given 访问登录页面
    And 输入手机号码 "<account>"
    And 输入密码 "<password>"
    When 点击登录按钮
    Then 跳转团队首页

  Examples:
    | account | password |
    | 135xxx1 | xxxxxxxx |
    | 135xxx2 | xxxxxxxx |

PO

页面对象(Page Object) 模式是 Selenium 实战中最为流行的一种设计模式, 在设计测试时, 常把页面元素定位和元素操作方法按照页面抽象出来, 分离成一定的对象, 再进行重新组织

在实际自动化测试过程中, 一般将脚本的实现分为 4 层:

  • 基础层: 用于存放 driver 对象及初始化使用;

  • 对象层: 用于存放页面元素定位和控件操作行为;

  • 操作层: 用于封装各个模块功能用例的操作;

  • 业务层: 真正执行测试用例的操作部分

示例

/* 示例: 基础层 */
public class BasePage {
    private static WebDriver driver;
    public BasePage() {
        System.setProperty("webdriver.edge.driver", "driver/msedgedriver.exe");
        driver = new EdgeDriver();
        driver.manage().window().maximize();
        driver.manage().timeouts().pageLoadTimeout(5L, TimeUnit.SECONDS);
    }
    public static WebDriver driver() {
        return driver;
    }
    public void open() {
        driver.get("http://news.anhuai.com/user/login");
    }
    public void close() {
        driver.quit();
    }
    public LoginPage login() {
        return new LoginPage();
    }
}
/* 示例: 对象层 */
public class LoginPage {
    public LoginPage() {
        PageFactory.initElements(BasePage.driver(), this);
    }
    @FindBy(id = "username")
    @CacheLookup
    private WebElement username;
    @FindBy(id = "password")
    @CacheLookup
    private WebElement password;
    @FindBy(className = "ant-btn")
    @CacheLookup
    private WebElement submit;
    @FindBy(className = "ant-form-item-explain")
    private Set<WebElement> message;
    public void sendUsername(String username) {
        this.username.clear();
        this.username.sendKeys(username);
    }
    public void sendPassword(String password) {
        this.password.clear();
        this.password.sendKeys(password);
    }
    public boolean submit() {
        this.submit.click();
        return CollectionUtils.isEmpty(this.message);
    }
}
/* 示例: 操作层 */
public class LoginMovies {
    public void login(String username, String password) {
        BasePage base = new BasePage();
        base.open();
        base.login().sendUsername(username);
        base.login().sendPassword(password);
        boolean submit = base.login().submit();
        base.close();
        Assertions.assertTrue(submit);
    }
}
/* 示例: 业务层 */
public class PageTest {
    @Test
    public void login() {
        LoginMovies movies = new LoginMovies();
        movies.login("135xxxxxxxx", "xxxxxxxx");
    }
}
