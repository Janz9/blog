Prometheus API

发表于 2021-07-01 更新于 2021-07-03 分类于 运维 阅读次数：
本文字数： 21k 阅读时长 ≈ 19 分钟

API

Prometheus 控制台的对象信息查询和表达式查询, 是通过服务端提供的基于 http 协议的 api 接口实现的, 当前稳定的接口可以在服务端的 /api/v1/ 路由端点下访问, 详见 http-api

接口响应格式基于 Json 形式, 每个成功的请求都会返回一个 2xx 的状态码, 可能返回的其它状态码如下:

  • 400 参数缺少或不正确时
  • 422 表达式不能被执行时
  • 503 服务超时或中止时
// 响应体样例
{
    "status": "success"|"error",
    "data": <data>,
    // 当status=error时返回错误信息
    "errorType": "<string>",
    "error": "<string>",
    // 如果存在不限制请求执行的错误时, 可能返回警告数组
    "warnings": ["<string>"]
}

表达式

表达式查询分为瞬时和区间 2 个接口, 详见 expression-queries

  1. 瞬时查询接口, 用于在单个时间点计算表达式的结果

    GET/POST /api/v1/query
# 入参
query   = <string>                  # 查询的表达式
time    = <rfc3999|unix_timestamp>  # 可选, 查询的时间戳
timeout = <duration>                # 可选, 超时的时间
# 出参
resultType = "matrix"|"vector"|"scalar"|"string"
result     = <value>

  2. 区间查询接口, 用于在一段时间内计算表达式的结果

    GET/POST /api/v1/query_range
# 入参
query   = <string>                  # 查询的表达式
start   = <rfc3999|unix_timestamp>  # 起始时间戳
end     = <rfc3999|unix_timestamp>  # 结束时间戳
step    = <duration|float>          # 查询的步宽
timeout = <duration>                # 可选, 超时的时间
# 出参
resultType = "matrix"
result     = <value>

当表达式查询成功时, 对应 PromQL 计算结果的 4 种类型, 响应体中 data 字段的返回值如下:

  • 区间向量(range vectors)

    {
    "resultType": "matrix",
    "result": [
        {
            "metric": {
                "<label_name>": "<label_value>", ...
            },
            "values": [
                [ [<unix_time>, "<sample_value>"], ... ]
            ]
        }, ...
    ]
}

  • 瞬时向量(instant vectors)

    {
    "resultType": "vector",
    "result": [
        {
            "metric": {
                "<label_name>": "<label_value>", ...
            },
            "values": [
                <unix_time>, "<sample_value>"
            ]
        }, ...
    ]
}

  • 标量(scalars)

    {
    "resultType": "scalar",
    "result": [ <unix_time>, "<scalar_value>" ]
}

  • 字符串(strings)

    {
    "resultType": "string",
    "result": [ <unix_time>, "<string_value>" ]
}

指标

指标元数据查询接口, 用于查询指标的类型和描述信息, 详见 querying-metric-metadata

GET /api/v1/metadata
# 入参
metric = <string>  # 指定指标的名称
limit  = <number>  # 返回指标列表的最大数量

元数据查询有 3 个接口, 用于查询指标, 标签和标签值, 详见 querying-metadata

  1. 查询指标, 用于查询指标的标签和标签值

    GET/POST /api/v1/series
# 入参
match[] = <series_selector>  # 标签选择器, 必须至少提供一个标签名称
start   = <rfc3999|unix_timestamp>
end     = <rfc3999|unix_timestamp>

  2. 查询标签

    GET/POST /api/v1/labels
# 入参
match[] = <series_selector>
start   = <rfc3999|unix_timestamp>
end     = <rfc3999|unix_timestamp>

  3. 查询标签值

    GET/POST /api/v1/label/<label_name>/values
# 入参
match[] = <series_selector>
start   = <rfc3999|unix_timestamp>
end     = <rfc3999|unix_timestamp>

目标

目标元数据查询接口, 用于获取目标和其指标的元数据, 详见 querying-target-metadata

GET /api/v1/targets/metadata
# 入参
metric       = <string>           # 指定指标的名称, 如果为空则返回目标下的所有指标
match_target = <label_selectors>  # 标签选择器, 提供标签和标签值
limit        = <number>           # 返回的目标列表的最大数量

目标查询接口, 用于获取目标的配置信息和状态, 详见 targets

GET /api/v1/targets
# 入参
state = "any"|"active"|"dropped"  # 指定目标的状态, 默认any返回全部结果

规则

规则查询接口, 用于获取当前加载的记录规则和告警规则, 详见 rules

GET /api/v1/rules
# 入参
type = "alert"|"record"  # 指定规则的类型, 默认为空不做过滤

告警查询接口, 用于获取当前状态为活动的告警列表, 详见 alerts

GET /api/v1/alerts

告警管理器接口, 用于获取当前配置的告警管理器列表及其状态, 详见 alertmanagers

GET /api/v1/alertmanagers

状态

状态查询有 6 个接口, 用于获取当前 Prometheus 的配置, 详见 status

  1. 配置接口, 用于获取当前的配置信息

    GET /api/v1/status/config

  2. 标志接口, 用于获取 Prometheus 配置的标志值

    GET /api/v1/status/flags

  3. 运行时接口, 用于获取服务的各种运行时信息

    GET /api/v1/status/runtimeinfo

  4. 构建接口, 用于获取服务构建时的信息属性

    GET /api/v1/status/buildinfo

  5. TSDB 统计接口, 用于获取 Prometheus TSDB 的各种基数统计信息

    • headStats 提供以下统计数据
      • numSeries 时序的数量
      • chunkCount 块的数量
      • minTime 当前最小的时间戳
      • maxTime 当前最大的时间戳
    • seriesCountByMetricName 提供基于指标名称的时序数量
    • labelValueCountByLabelName 提供基于标签名称的标签值数量
    • memoryInBytesByLabelName 提供基于标签名称的内存使用量(单位字节)
    • seriesCountByLabelPair 提供基于标签键值对的时序数量
    GET /api/v1/status/tsdb

  6. WAL 重载统计接口, 用于获取预写入数据恢复的次数

    GET /api/v1/status/walreplay

管理员

管理员接口对运维和开发人员非常实用, 它给用户开放了时序数据库 TSDB 的操作权限, 详见 tsdb-admin-apis

# 示例: 管理员接口默认不启用, 需要在启动服务时指定选项
$ prometheus --web.enable-admin-api

管理员接口有 3 个可操作的端口

  1. snapshot 快照接口将当前所有数据在 tsdb 目录下的 snapshots/<datetime>-<rand> 文件夹内创建快照, 并将文件夹作为响应中的一个字段值返回

    POST/PUT /api/v1/admin/tsdb/snapshot
# 入参
skip_head = <bool>  # 可选, 是否跳过块中未压缩到磁盘的数据

  2. delete series 删除时序接口可以将指定的时序标记为删除, 并在下次 tsdb 执行定期清理操作时移除

    POST/PUT /api/v1/admin/tsdb/delete_series
# 入参
match[] = <series_selector>         # 标签选择器, 选择要删除的时序
start   = <rfc3999|unix_timestamp>
end     = <rfc3999|unix_timestamp>

  3. clean tombstones 清理里程碑接口将从磁盘中移除标记为删除的数据

    POST/PUT /api/v1/admin/tsdb/clean_tombstones
