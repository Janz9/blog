Prometheus API

发表于 2021-07-01 更新于 2021-07-03 分类于 运维 阅读次数： 本文字数： 22k 阅读时长 ≈ 20 分钟

API

Prometheus 控制台的对象信息查询和表达式查询, 是通过服务端提供的基于 http 协议的 api 接口实现的, 当前稳定的接口可以在服务端的 /api/v1/ 路由端点下访问, 详见 http-api

接口响应格式基于 Json 形式, 每个成功的请求都会返回一个 2xx 的状态码, 可能返回的其它状态码如下:

  • 400 参数缺少或不正确时
  • 422 表达式不能被执行时
  • 503 服务超时或中止时

表达式

表达式查询分为瞬时和区间 2 个接口, 详见 expression-queries

  1. 瞬时查询接口, 用于在单个时间点计算表达式的结果

  2. 区间查询接口, 用于在一段时间内计算表达式的结果

当表达式查询成功时, 对应 PromQL 计算结果的 4 种类型, 响应体中 data 字段的返回值如下:

  • 区间向量(range vectors)

  • 瞬时向量(instant vectors)

  • 标量(scalars)

  • 字符串(strings)

指标

指标元数据查询接口, 用于查询指标的类型和描述信息, 详见 querying-metric-metadata

元数据查询有 3 个接口, 用于查询指标, 标签和标签值, 详见 querying-metadata

  1. 查询指标, 用于查询指标的标签和标签值

  2. 查询标签

  3. 查询标签值

目标

目标元数据查询接口, 用于获取目标和其指标的元数据, 详见 querying-target-metadata

目标查询接口, 用于获取目标的配置信息和状态, 详见 targets

规则

规则查询接口, 用于获取当前加载的记录规则和告警规则, 详见 rules

告警查询接口, 用于获取当前状态为活动的告警列表, 详见 alerts

告警管理器接口, 用于获取当前配置的告警管理器列表及其状态, 详见 alertmanagers

状态

状态查询有 6 个接口, 用于获取当前 Prometheus 的配置, 详见 status

  1. 配置接口, 用于获取当前的配置信息

  2. 标志接口, 用于获取 Prometheus 配置的标志值

  3. 运行时接口, 用于获取服务的各种运行时信息

  4. 构建接口, 用于获取服务构建时的信息属性

  5. TSDB 统计接口, 用于获取 Prometheus TSDB 的各种基数统计信息

    • headStats 提供以下统计数据
      • numSeries 时序的数量
      • chunkCount 块的数量
      • minTime 当前最小的时间戳
      • maxTime 当前最大的时间戳
    • seriesCountByMetricName 提供基于指标名称的时序数量
    • labelValueCountByLabelName 提供基于标签名称的标签值数量
    • memoryInBytesByLabelName 提供基于标签名称的内存使用量(单位字节)
    • seriesCountByLabelPair 提供基于标签键值对的时序数量

  6. WAL 重载统计接口, 用于获取预写入数据恢复的次数

管理员

管理员接口对运维和开发人员非常实用, 它给用户开放了时序数据库 TSDB 的操作权限, 详见 tsdb-admin-apis

管理员接口有 3 个可操作的端口

  1. snapshot 快照接口将当前所有数据在 tsdb 目录下的 snapshots/<datetime>-<rand> 文件夹内创建快照, 并将文件夹作为响应中的一个字段值返回

  2. delete series 删除时序接口可以将指定的时序标记为删除, 并在下次 tsdb 执行定期清理操作时移除

  3. clean tombstones 清理里程碑接口将从磁盘中移除标记为删除的数据
