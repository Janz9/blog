Micrometer

Micrometer 是基于 JVM 应用的度量监控库, 它提供了应用内统一的指标采集接口和实现, 允许开发人员选择不同的数据规范对接方式, 从而最大限度的提高了对接不同监控系统的可移植性

由于不同的监控系统对数据规范的特征支持不一, Micrometer 按 3 个特征对已支持的监控系统做了划分, 详见 monitoring-systems

  • 维度(dimensional) 对于支持维度的监控系统, 维度信息将以键值对的标签形式返回, 对于不支持维度的监控系统, 维度信息将会展平添加到指标名称中
  • 速率聚合(rate aggregation) 对于不支持速率聚合的监控系统, 增量采样的数据可以进行速率聚合再返回新样本, 对于支持速率聚合的监控系统, 直接返回增量数据
  • 发布(publishing) 支持不同的监控系统对采样数据的推模式和拉模式

注册表

Micrometer 为不同的监控系统提供了称为 注册表(registry) 的实现模块和测试套件, 注册表接口 MeterRegistry 定义了创建和保存应用要采集的指标元数据的方法, 每个支持的监控系统都有一个该接口的具体实现, 详见 registry

Micrometer 默认提供 SimpleMeterRegistry 的简单实现类, 在开发者没有首选的监控系统前, 可以通过该实现类编写采样的代码, 该实现类将采样数据保存在内存中, 并且不会提供对外暴露对接数据的方式

/* 示例: registry */
SimpleMeterRegistry registry = new SimpleMeterRegistry();  // 创建一个简单的注册表
Meter metrics = registry.counter("biz.metrics");           // 创建一个counter类型的指标

过滤器

注册表可以通过配置指标过滤器, 控制指标的注册方式和其采样数据, 详见 meter-filters

过滤器有三个基本功能:

  1. 禁用已注册的指标

    /* 示例 */
registry.config()
    .meterFilter(MeterFilter.ignoreTags("jvm"))
    .meterFilter(MeterFilter.denyNameStartsWith("jvm"));

  2. 转换指标的 Id 对象, 其包含指标名称和标签集

    /* 示例 */
registry.config()
    .meterFilter(new MeterFilter() {
        @Override
        public Meter.Id map(Meter.Id id) {
            if(id.getName().startsWith("test")) {
                return id.withName("extra." + id.getName()).withTag(Tag.of("extra.tag", "value"));
            }
            return id;
        }
    });

  3. 为指定类型的指标配置分布统计

    /* 示例 */
registry.config()
    .meterFilter(new MeterFilter() {
        @Override
        public DistributionStatisticConfig configure(Meter.Id id, DistributionStatisticConfig config) {
            if (id.getName().startsWith("prefix")) {
                return DistributionStatisticConfig.builder()
                    .serviceLevelObjectives(0.9, 0.95)
                    .build()
                    .merge(config);
            }
            return config;
        }
    });

指标

Micrometer 定义了可以采集的指标接口 Meter, 并提供了以下原生的指标类型, 详见 meter

  • Counter 单调递增且必须为正数的指标
  • Gauge 实时变化的数值的指标
  • DistributionSummary 直方图和分位图的指标
  • Timer 短时间任务的总次数和总时间的指标
  • LongTaskTimer 长时间任务的当前次数和已用时的指标
  • FunctionCounter FunctionTimer TimeGauge

Micrometer 为了一致性和可移植性的原因, 指标名称的生成方案与 Prometheus 不完全一致, 如果希望使用 Prometheus 标准的指标名称, 可以通过配置重命名过滤器

/* 示例: 配置过滤器 */
registry.config()
    .meterFilter(new PrometheusRenameFilter());

Counter

计数指标通常可以通过速率聚合的运算得到在一段时间内的使用率, 需要注意当应用重启时, 使用内存保存数据的计数指标会重置从零开始递增, 这可能会导致速率运算的结果错误, 以致于表达式配置的告警在很长时间内发出异常行为, 详见 counter

/* 示例: 实例 */
Counter counter = Counter.builder("biz.counter")
    .tag("event", "login")           // 标签
    .description("Counter of biz.")  // 描述
    .register(registry);
// 递增值
counter.increment();
counter.increment(double amount);

当使用 Prometheus 做速率运算时, 速率函数会自动检测计数指标的数据是否重置计数, 从而避免速率运算的结果错误, 因此在执行速率聚合的表达式时, 都应该先执行速率后执行聚合, 如 sum(rate())

Gauge

测量指标可以用于跟踪任何可设置的 java.lang.Number 子类型对象, 与其它类型的指标不同, 测量型指标不直接对指标的引用进行操作, 而是通过赋值待测量的对象数值即可, 详见 gauge

/* 示例: 实例 */
AtomicInteger number = new AtomicInteger(0);
Gauge gauge = Gauge.builder("biz.gauge", this::getNumber)
    .description("Gauge of biz.")
    .register(registry);
// 测量值
number.set(int newValue);

DistributionSummary

分布摘要指标用于跟踪事件的分布, 可通过配置用于直方图和分位图, 详见 distribution-summary

/* 示例: 实例 */
DistributionSummary histogram = DistributionSummary.builder("biz.histogram")
    .description("Histogram of biz.")
    .serviceLevelObjectives(25, 50, 90)  // 直方图的上边界
    .minimumExpectedValue(1d)            // 预期最小采样
    .maximumExpectedValue(100d)          // 预期最大采样
    .register(registry);
DistributionSummary summary = DistributionSummary.builder("biz.summary")
    .description("Summary of biz.")
    .publishPercentiles(0.25, 0.5, 0.95)  // 分布图的分位数
    .minimumExpectedValue(1d)
    .maximumExpectedValue(100d)
    .register(registry);
// 记录采样值
histogram.record(double amount);
summary.record(double amount);

Timer

计时指标用于测量短任务的总执行时间和执行频率, 计时统计的总时间的最大值为 Long.MAX_VALUE (约 292.3 年), 计时的基本单位因记录时的度量而异, 详见 timer

/* 示例: 实例 */
Timer timer = Timer.builder("biz.timer")
    .description("Timer of biz.")
    .register(registry);
// 记录指定时间
timer.record(Duration duration);
// 通过执行记录时间
Timer.Sample = Timer.start();
sample.stop(timer);

LongTaskTimer

长任务计时指标用于测量仍在运行中的任务的已执行时间和次数, 当任务执行完成后, 样本不会再计入统计的指标, 详见 long-task-timer

/* 示例: 实例 */
LongTaskTimer longTaskTimer = LongTaskTimer.builder("biz.longTaskTimer")
    .description("LongTaskTimer of biz.")
    .register(meterRegistry);
// 记录任务时间
LongTaskTimer.Sample sample = longTaskTimer.start();
sample.stop();

示例

SpringBoot 2.x 版本的 spring-boot-actuator 引入了 Micrometer 作为默认的监控系统实现方案, 详见 actuator-metrics

<!-- 示例: 依赖配置 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
<dependency>  <!-- 此处使用prometheus作为监控系统 -->
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
<dependency>  <!-- 可选扩展 -->
    <groupId>io.github.mweirauch</groupId>
    <artifactId>micrometer-jvm-extras</artifactId>
</dependency>
# 文件: application.yml
spring:
  application:
    name: myapp
management:
  metrics:
    tags:  # 为采集的指标都添加的标签集
      application: ${spring.application.name}
    export:
      simple:  # 禁用默认的简单注册表实例
        enabled: false
      prometheus:  # 启用Prometheus的注册表实例
        enabled: true

注解

Micrometer 提供了注解的方式, 为特定类型的方法添加时序监控的支持, 默认 SpringBoot 的自动配置中, 注解只对特定的方法 (比如 Controller 的请求映射函数等) 有效, 可以通过配置切面实现注解对所有方法的支持, 详见 annotation

/* 示例: 配置 */
@Configuration
public class TimedConfiguration {
    @Bean  // 配置此切面实例, 则时序监控的注解对所有方法都可用
    public TimedAspect timedAspect(MeterRegistry registry) {
        return new TimedAspect(registry);
    }
}
/* 示例: 注解 */
@Timed(value = "biz.request")  // 统计请求的时序
@GetMapping("/hello")
public String hello() {
    return "Hello World";
}
@Counted(value = "biz.task.failure", recordFailuresOnly = true)  // 统计任务失败的时序
@Timed(value = "biz.task", longTask = true)                      // 统计任务执行的时序
@Scheduled(fixedDelay = 60000L)  // 每隔60s定时执行
public void task() throws InterruptedException {
    Thread.sleep(5000L);  // 模拟任务执行5s的用时
}

业务

Micrometer 监控业务指标主要涉及业务层, 可能需要根据业务流程中的中间数据作为指标的标签集或者采样依据, 为了避免硬编码导致业务代码复杂化, 通常可以通过切面或监听器等外部配置来描述业务指标的采集逻辑

/* 示例: AOP配置采集业务指标 */
@Aspect
@Component
public class BizAspect implements InitializingBean {

    @Autowired
    private MeterRegistry registry;

    private static Counter bizOrder;
    private static DistributionSummary bizOrderTime;

    @AfterReturning(value = "execution(* com.janz.micrometer.OrderBiz.save(..))", returning = "order")
    public void save(OrderDO order) {
        bizOrder.increment();
        bizOrderTime.record(Calendar.getInstance().get(Calendar.HOUR_OF_DAY));
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        // 统计下单的总数
        bizOrder = Counter.builder("biz.order")
            .description("Total number of orders.")
            .register(registry);
        // 统计按时段下单的分布
        bizOrderTime = DistributionSummary.builder("biz.order.time")
            .description("Time distribution of orders.")
            .serviceLevelObjectives(6d, 12d, 15d, 20d, 24d)
            .publishPercentiles(0.5d, 0.95d)
            .minimumExpectedValue(0.1d)
            .maximumExpectedValue(24d)
            .register(registry);
    }
}
