Vim 定制

发表于 2021-04-25 更新于 2021-04-30 分类于 工具 阅读次数： 本文字数： 6.1k 阅读时长 ≈ 6 分钟

定制

用户界面

Vim 用户界面是可扩展的, 用户可以改变 Vim 的主题, 修改某些界面元素的显示方式, 并且增强状态栏中的信息显示

配色

Vim 中可以使用多种配色, 一部分是 Vim 自带的, 一部分则由社区成员提供

Colorschemes 提供了大量的 Vim 配色

状态栏

状态栏是屏幕底部用于显示信息的一个区域

Airline 插件提供了轻量级, 信息足够丰富的状态栏

同步

一般而言, 文件的同步方式有很多, 用户通常会将文件存储在一个 Git 仓库中, 这些配置文件常常称为点文件(dotfiles), 并在主目录中用符号链接文件指向点文件目录中的相应文件

Fugitive 插件可以使用户在不离开 Vim 的情况下进行 Git 操作, 用户在使用 Vim 编辑文件时, 也可以同时利用版本管理系统记录编辑的位置

Vimscript

Vimscript 是一门用于定制 Vim 的脚本语言, 脚本的命令都可在 Vim 的命令行模式下运行, 习惯上 Vimscript 都采用后缀名 .vim

变量

Vim 中变量和函数的作用域是通过前缀实现的

整数变量之间可以进行常用的加减乘除, 字符串拼接使用点运算符

Vim 支持更复杂的数据结构, 比如列表和字典

条件

条件表达式支持常用的 与&&, 或||, 非! 逻辑运算符, 因为文本编辑本质是字符串操作, Vim 提供了一些文本比较运算符:

  • == 字符串匹配
  • =~ 正则匹配
  • !~ 正则不匹配

检查当前正在运行的 Vim 是否支持某个功能

循环

循环控制流支持用 continue 和 break 来控制流程

函数

在 Vim 中, 定义的函数必须以大写字母开头, 除非函数在脚本作用域或命名空间中, 否则会报错

Vim 脚本可能会被加载多次, 但函数不一定会被重加载(比如修改参数个数时), 解决方案是使用 function! 来定义函数, 加载时强制使用最新的函数定义, 这在测试时非常有用

Lambda

Lambda 表达式是一种匿名函数, 它有助于编写逻辑清晰的代码

Vim 支持将函数作为参数传递给另一个函数

类

Vim 本身没有类, 但是它支持在字典上定义方法, 因此可以实现面向对象编程范式, 有两种方式可以在字典上定义方法

命令

Vim 提供了几个常用的命令, 用于将参数作为执行命令

  • execute 将参数解析为Vim命令并执行
  • normal 将参数解析为Vim按键序列(与正常模式的操作相似)
  • silent 用于隐藏命令的输出

Vim 文本编辑器的大部分操作是针对文件的, 它提供了许多文件相关的函数

  • expand 用于操作文件路径信息
  • filereadable 用于检查文件是否可读(是否存在)
  • filewritable 用于检查文件是否可写(检查权限)

交互

Vim 中有两种方式提示用户输入:

  • confirm 显示多选对话框
  • input 自定义复杂的输入

如果用户在按键重映射中使用了 input 函数, 则必须先调用 inputsave(), 并在之后调用 inputrestore(), 否则映射中剩余字符会被 input 函数获取

插件

Vim8 的插件布局应该包含如下目录结构, 它与主流插件管理器兼容

  • autoload 用于保存插件延迟加载的内容
  • colors 用于保存配色
  • compiler 用于保存编译器相关的功能
  • doc 文档目录
  • ftdetect 用于保存文件类型检测设置
  • ftplugin 用于保存文件类型相关的代码
  • indent 用于保存缩进相关的设置
  • plugin 用于保存插件的核心功能
  • syntax 用于保存语言语法分组

示例

示例实现一个简单的功能, 即添加一个按键绑定, 用于在当前行的行首添加 Java 风格的注释符号
