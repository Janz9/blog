Vim 定制

发表于 2021-04-25 更新于 2021-04-30 分类于 工具 阅读次数：
本文字数： 6k 阅读时长 ≈ 5 分钟

定制

用户界面

Vim 用户界面是可扩展的, 用户可以改变 Vim 的主题, 修改某些界面元素的显示方式, 并且增强状态栏中的信息显示

配色

Vim 中可以使用多种配色, 一部分是 Vim 自带的, 一部分则由社区成员提供

" 示例: .vimrc配置
set background=dark  " 设置背景色
colorscheme murphy   " 设置主题

Colorschemes 提供了大量的 Vim 配色

状态栏

状态栏是屏幕底部用于显示信息的一个区域

" 示例: .vimrc配置
set laststatus=2  " 总显示状态栏
set showcmd       " 在状态栏中显示最后执行的命令

Airline 插件提供了轻量级, 信息足够丰富的状态栏

同步

一般而言, 文件的同步方式有很多, 用户通常会将文件存储在一个 Git 仓库中, 这些配置文件常常称为点文件(dotfiles), 并在主目录中用符号链接文件指向点文件目录中的相应文件

# 示例: Git同步配置
> mkdir ~/.dotfiles              # 创建点文件目录
> git clone <URL> ~/.dotfiles    # 克隆版本库
> ln -s ~/.dotfiles/.vim ~/.vim  # 创建符号链接

Fugitive 插件可以使用户在不离开 Vim 的情况下进行 Git 操作, 用户在使用 Vim 编辑文件时, 也可以同时利用版本管理系统记录编辑的位置

Vimscript

Vimscript 是一门用于定制 Vim 的脚本语言, 脚本的命令都可在 Vim 的命令行模式下运行, 习惯上 Vimscript 都采用后缀名 .vim

# 示例: 运行Vim脚本文件
:source xxx.vim
:source %  # 运行当前文件
# 示例: 查看Vim脚本相关的帮助文档
:help eval

变量

Vim 中变量和函数的作用域是通过前缀实现的

" 示例: 变量
set background=dark         " set关键字为内部选项赋值
let g:airline_theme='term'  " let关键字为非内部变量赋值
let &ignorecase=0           " let赋值Vim选项(布尔1真0假)
let @a=''                   " let赋值寄存器(寄存器a)

# 常用的作用域
g  # 全局作用域(若未指定作用域, 则默认为全局作用域)
v  # 为Vim所定义的全局作用域
l  # 局部作用域(在函数内部, 若未指定作用域, 则默认为局部作用域)
b  # 表示当前缓冲区
w  # 表示当前窗口
t  # 表示当前标签页
a  # 表示函数的参数

整数变量之间可以进行常用的加减乘除, 字符串拼接使用点运算符

" 示例: 点运算符
let b:message='theme: '.g:airline_theme
echo b:message  " 打印变量内容到状态栏中

Vim 支持更复杂的数据结构, 比如列表和字典

" 示例: 数据结构
let b:lan = ['zh', 'en', 'fr']  " 列表
let b:lan2 = {                  " 字典
  \ 'zh': '中文',               " 代码跨多行则需要用反斜划线表示换行
  \ 'en': '英文',
  \ 'fr': '法文'
}

条件

" 示例: 条件
if b:value == 'zh'
  echo '中文'
elseif b:value == 'en'
  echo '英文'
else
  echo '其它'
endif
" 三元运算符
echo b:value=='zh'?'中文':'其它'

条件表达式支持常用的 与&&, 或||, 非! 逻辑运算符, 因为文本编辑本质是字符串操作, Vim 提供了一些文本比较运算符:

  • == 字符串匹配
  • =~ 正则匹配
  • !~ 正则不匹配
" 示例: 逻辑运算符
'zh' == 'ZH'          " 是否忽略大小写的默认行为取决于ignorecase的设置
'zh-cn' =~ '\vzh\w*'  " 真
'zh-cn' !~ '\v\W+'    " 真

" 在逻辑运算符后接?强制忽略
" 在逻辑运算符后接#强制考虑
'zh' ==? 'ZH'         " 真
'zh' ==# 'ZH'         " 假
'zh-cn' !~# '\vzh.*'  " 假

检查当前正在运行的 Vim 是否支持某个功能

" 示例: 判断功能
if has('python3')               " Vim提供了几个系统标示符, 它们对于编写跨平台的脚本非常有用
  let g:gundo_prefer_python3=1  " 当前Vim支持python3时配置插件
endif
# 示例: 查看完整的功能列表
:help feature-list

循环

循环控制流支持用 continue 和 break 来控制流程

" 示例: 循环
for l in lan              " 循环列表
  echo 'value: '.l
endfor

for k in keys(lan2)       " 循环字典
  echo 'key: '.k.', value: '.lan2[k]
endfor

for [k,v] in items(lan2)  " 循环字典
  if k == 'zh'
    break                 " 跳出
  else
    continue              " 继续
  endif
  echo 'key: '.k.', value: '.v
endfor

函数

在 Vim 中, 定义的函数必须以大写字母开头, 除非函数在脚本作用域或命名空间中, 否则会报错

" 示例: 定义函数
function Translate(lan, ...)               " 固定参数和可变数目参数可以结合使用
  echo 'value: '.a:lan                     " 在函数内访问参数时需要使用a:作用域
  echo 'values: '.join(a:000, ', ')        " a:000获取可变数目参数的列表, join是内置的列表操作函数
  echo 'values: '.a:lan.', '.a:1.', '.a:2  " 通过序号获取可变数目参数的单个参数
endfunction
# 示例: 调用函数
:call Translate('zh', 'en', 'fr')  " 调用
# 状态栏输出
value: zh
values: en, fr
values: zh, en, fr
请按 ENTER 或其它命令继续

Vim 脚本可能会被加载多次, 但函数不一定会被重加载(比如修改参数个数时), 解决方案是使用 function! 来定义函数, 加载时强制使用最新的函数定义, 这在测试时非常有用

Lambda

Lambda 表达式是一种匿名函数, 它有助于编写逻辑清晰的代码

" 示例: Lambda表达式
let b:Func = {arg -> 'arg: '.arg}

# 调用
:echo b:Func('zh')

Vim 支持将函数作为参数传递给另一个函数

" 示例: 函数传递
function! Echo(frag, func)  " 第二个参数为函数引用
  echo a:func(a:frag, '-')
endfunction

# 调用
:call Echo(['zh', 'cn'], function('join'))                     # function引入一个函数
:call Echo(['zh', 'cn'], {frag,flag -> frag[0].flag.frag[1]})  # 匿名函数

类

Vim 本身没有类, 但是它支持在字典上定义方法, 因此可以实现面向对象编程范式, 有两种方式可以在字典上定义方法

" 示例: 方法一
function b:lan2.Translate(key)                  " 添加函数
  return 'key: '.a:key.', value: '.self[a:key]  " self指向字典自身
endfunction
" 示例: 方法二
function Translate(key) dict                     " 函数名后添加dict关键字
  return 'key: '.a:key.', value: '.self[a:key]
endfunction
let b:lan2['Translate'] = function('Translate')  " 将函数绑定到字典的键上
# 示例: 调用
:echo b:lan2.Translate('zh')

命令

Vim 提供了几个常用的命令, 用于将参数作为执行命令

  • execute 将参数解析为Vim命令并执行
  • normal 将参数解析为Vim按键序列(与正常模式的操作相似)
  • silent 用于隐藏命令的输出
" 示例: 命令
execute 'echo &ignorecase'  " 同命令行模式执行:echo &ignorecase
normal dd                   " 同正常模式执行dd(删除当前行)
silent echo 'message'       " 执行并且隐藏输出

" normal执行的按键序列会遵守用户配置的按键重映射, 如果要忽略用户配置, 需要使用normal!指令

Vim 文本编辑器的大部分操作是针对文件的, 它提供了许多文件相关的函数

  • expand 用于操作文件路径信息
  • filereadable 用于检查文件是否可读(是否存在)
  • filewritable 用于检查文件是否可写(检查权限)
" 示例: 文件命令
if filereadable(expand('%'))
  echo 'current file ('.expand('%:t').') is readable.'
endif

# expand的参数是文件名, 可以用特殊符号(比如%表示当前文件), 文件名还可以包含以下修饰符
:p  # 表示展开为完整路径 
:h  # 表示路径头(路径最后一个分量被删除)
:t  # 表示路径尾(只保留路径最后一个分量)
:r  # 表示根路径(删除一个文件扩展)
:e  # 表示只保留文件扩展

交互

Vim 中有两种方式提示用户输入:

  • confirm 显示多选对话框
  • input 自定义复杂的输入
" 示例: confirm
let b:answer = confirm('quit?', "&yes\n&no")
echo b:answer

# 执行如下
:source %
quit?
[y]es, [n]o: # 此处等待用户输入, 输入y则输出1, 输入n则输出2
" 示例: input
let b:answer = input('what is your preferred language?')
echo "\n".b:answer

# 执行如下
:source %
what is your preferred language?  # 此处等待用户输入
# 此处输出用户的输入

如果用户在按键重映射中使用了 input 函数, 则必须先调用 inputsave(), 并在之后调用 inputrestore(), 否则映射中剩余字符会被 input 函数获取

" 示例: input
function! Func()
  call inputsave()
  let opera = input('what is your operation?')
  call inputrestore()
  return opera
endfunction
nnoremap <LEADER>o :let b:temp = Func()<CR>:echo b:temp<CR>

插件

Vim8 的插件布局应该包含如下目录结构, 它与主流插件管理器兼容

  • autoload 用于保存插件延迟加载的内容
  • colors 用于保存配色
  • compiler 用于保存编译器相关的功能
  • doc 文档目录
  • ftdetect 用于保存文件类型检测设置
  • ftplugin 用于保存文件类型相关的代码
  • indent 用于保存缩进相关的设置
  • plugin 用于保存插件的核心功能
  • syntax 用于保存语言语法分组

示例

示例实现一个简单的功能, 即添加一个按键绑定, 用于在当前行的行首添加 Java 风格的注释符号

# 示例: 创建项目
> mkdir -p ~/.vim/pack/plugins/start/vim-commenter
> touch ~/.vim/pack/plugins/start/vim-commenter/plugin/commenter.vim
" 文件: commenter.vim
function! g:commenter#Comment()
  let l:line = getline('.')       " 获取当前行的内容
  call setline('.', '//'.l:line)  " 在当前行的行首添加注释符号
endfunction
nnoremap gc :call g:commenter#Comment()<CR>  " 将函数映射到gc按键
# 示例: 
:source $MYVIMRC  # 重加载配置
gc                # 光标移到需要注释的行, 执行gc
