Vim 工作区

发表于 2021-04-24 更新于 2021-04-30 分类于 工具 阅读次数： 本文字数： 4.6k 阅读时长 ≈ 4 分钟

工作区

在编写程序时, 经常需要同时处理多个文件, 涉及来回切换, 跨文件编辑或到其他界面查询资料等操作, Vim 可以在工作中同时处理任意多个文件

缓冲区

缓冲区是文件的内部表示, 每个打开的文件都有一个缓冲区

窗口

Vim 将缓冲区加载到窗口中, 一个屏幕上可以同时显示多个窗口

使光标在不同窗口间移动, 先按 Ctrl+w 组合键, 再输入方向键

标签页

Vim 用标签页来组织一个窗口的集合, 进而支持在不同的窗口集合之间切换, 这让用户方便的拥有了多个工作区

文件树

Netrw 是 Vim 内置的文件管理器, 支持对文件和目录的浏览, 这和操作系统下的文件管理器类似

Netrw 的窗口也可以在分割窗口或标签页中打开

NERDTree 是一种模拟现代 IDE 行为的优秀插件, 它在屏幕边缘用一个分割的缓冲区来展示文件树, 并且支持书签

文件

Vim 拥有一些浏览复杂代码结构的功能

折叠

折叠功能支持文件部分内容的隐藏, 隐藏的依据既可以是预定义的规则, 也可以是手动添加的折叠标记, Vim 在折叠代码方面是比较智能的, 而且支持多种折叠方式

自动补全

Vim 内置了代码自动补全的功能, 通过自动补全, 用户可以在编辑时避免拼写错误, 而且无须反复输入和记忆冗长复杂的变量名

自动补全功能从 Vim7 开始默认启用, 输入函数名的开始几个字符, 然后按 Ctrl+n 组合键, Vim 会弹出一个自动补全列表, 可以通过 Ctrl+n 和 Ctrl+p 在这个列表中上下移动, 以选择合适的备选项, 如果继续输入, 列表则会消失

YouCompleteMe 插件扩展了 Vim 内置的自动补全引擎, 而且还增添了大量新功能:

  • 支持程序语言感知的自动语义补全;
  • 补全建议的智能排序和过滤;
  • 能够显示代码文档, 重命名变量, 自动格式化代码以及修正某些类别的输入错误

标签

Vim 自带了在同一文件中浏览变量定义的功能, 将光标置于一个单词上, 输入 gd 会优先查看变量的局部声明, 输入 gD 会查看全局声明

此功能并没有考虑语法, 因为 Vim 本身并不了解代码的语义结构, 不过, Vim 支持标签功能, 标签文件中可存储程序代码的语义和结构关键词, 供 Vim 检索

Exuberant Ctags 是一个用于生成标签文件的外部工具

撤销树

大部分现代编辑器都支持撤销栈, 其中保存了撤销和重做操作的历史记录, Vim 则更进一步, 采用了撤销树

Gundo 是一个对撤销树进行可视化的插件

执行 :GundoToggle 将以分割方式打开(或关闭)两个新窗口, 左上窗口为树状结构的可视化表达, 光标默认选中了最新的版本, 左下窗口则显示此版本和上一版本之间的差异, 按 Enter 键将恢复到选中的版本

差异

在开发过程中, 经常需要比较不同文件之间的差异, 比如比较不同的输出, 或者比较一个文件的不同版本, 或者处理合并时的冲突, Vim 提供了一个独立的可执行程序 vimdiff 用于对比文件差异

此命令会打开两个文件的窗口, 不同的行用一种颜色高亮显示, 而不同的字符则用另一种颜色高亮显示, 用户可以用 ]c 和 [c 分别在多个修改处之间向前和向后跳转

用户可以在两个文件之间推送修改:

  • do 或 :diffget 将文件修改应用于当前窗口中的文件
  • dp 或 :diffput 将当前窗口中的文件修改推送给另一个文件

若要将一个文件整体复制到另一个文件中, 可以使用 :%diffget 或 :%diffput, 当用户手动修改了文件, 则需要执行 :diffupdate 手动更新高亮显示

参数列表

参数列表(argument list, arglist) 支持在多个文件中执行同一操作, 而不需要用户预先加载缓冲区

参数列表支持如下命令:

  • arg 用于定义参数列表
  • argdo 对参数列表中的所有文件执行一条命令
  • args 用于显示参数列表中的文件列表

正则表达式

正则表达式(regular expression) 是对字符串操作的一种逻辑公式, Vim 有一套独特的正则表达式语法, 和其他很多正则表达式不同, Vim 中的正则表达式的特殊字符需要用 \ 来转义

Vim 通过 :substitute 命令实现搜索和替换功能, 简写为 :s

区间范围

:substitute 命令可以作用于一个区间范围, 即指定范围的内容中的匹配项会被替换, 常用的范围是 %, 即作用于整个文件, 区间范围可以用 ; 运算符组合起来

魔法模式

当用户需要编写较长的正则表达式时, 对每个特殊字符都使用 \ 转义就比较繁琐, 这时候就需要用到 Vim 的魔法模式了, Vim 的魔法模式用于确定如何解析正则表达式, 有 3 种模式:

  • 基本魔法(magic) Vim 默认模式, 大部分特殊字符都需要转义, 少数除外如 . *

  • 无魔法(no magic) 类似于基本魔法, 只不过每个特殊字符都需要转义, 包括 . *

  • 深度魔法(very magic) 深度魔法将数字, 字母和下划线之外的字符都视为特殊字符, 不再转义

宏

宏(macro) 支持录制一系列动作, 然后用于回放, Vim 用命令 q 进入宏录制模式, 后面接任意寄存器

递归

在命令前加数字表示多次重复运行宏, 不过更好的做法是, Vim 支持递归的宏

终端

Vim 中运行 shell 命令可以使用 :! 命令, 这种方式不太方便, 因为用户无法在运行程序的同时编辑文档

Vim8.1 版本中引入终端模式, 终端模式是 Vim 会话中运行的一个终端模拟器, 当用户执行终端命令时, 可以同时使用 Vim 进行其他操作

Tmux

Tmux 是一个终端复用工具, 可以在终端屏幕中管理多个终端窗口, Tmux 面板和 Vim 窗口可以互补, 用户可以在不同的 Tmux 面板中打开 Vim, 从而让这些实例相互独立, 实现缓冲区的分组
