Vim 简介

发表于 2021-04-24 更新于 2021-04-30 分类于 工具 阅读次数： 本文字数： 5.5k 阅读时长 ≈ 5 分钟

简介

Vim 是一个高度可配置的文本编辑器, 旨在非常高效的创建和更改任何类型的文本

原来的 Vi 版本是针对远程终端开发出来的, 那时的带宽和网速都有限, 但正是这些限制使 Vi 的文本编辑流程变得高效而专业, 从而演变为如今的 Vim(Vi Improved) 的核心

安装

Vim 可在各种操作系统中安装, 官网 download 提供了详细的安装说明

需要注意的是, 包管理器软件仓库中的 Vim 版本一般会有所滞后, 安装 Vim 的最佳方式是使用源代码, 这需要编译器及其支持的文件

手册

Vim 提供了一个学习工具 :help 命令, Vim 手册中附带了大量的资源和教程

为找到正确的词条, 可以在输入指令 :help search 时通过 Ctrl+d 组合键得到包含 search 的标签列表

配置

Vim 是一个高可配置软件, 支持不同层次的配置文件, 详见 settings

windows 系统不允许文件名中出现句号, 因此配置文件的名称为 _vimrc

如果遇到了问题, 可以在 Vim 中执行 :echo $MYVIMRC 指令, Vim 会显示它正在使用的 .vimrc 路径

常用配置

可选配置参考 options

重映射

Vim 支持将每个操作的按键映射为其他键, 创建一个新映射时, 最好先确认一下此键或按键序列是否已经被映射为其他用途了, 内置的按键绑定列表可通过 :help index 命令查看

Vim 支持细粒度的映射控制方式, 可以针对每种模式分别定义映射

先导键

Vim 常用先导键来实现额外的快捷键映射, 默认情况下, 先导键为反斜线 \, 这个键使用起来不太方便, 在 Vim 社区中有几个替代的方案, 其中逗号 , 是比较流行的

插件

Vim 的功能可以通过添加插件来扩展, Vim8 提供了一种原生的插件加载方式:

  • .vim/pack/<folder>/opt/ 保存的插件用于手动加载
  • .vim/pack/<folder>/start 保存的插件用于始终加载

Vim 的插件管理方式一直在与时俱进, 网上也能找到很多插件管理器, vim-plug 是很有前景的插件管理器, 这个插件管理器是轻量级的, 易用且兼容大量的插件

插件配置

Vim 插件通常会提供命令来绑定快捷键, 或提供变量用于控制插件的行为, 因此, 在使用一个插件时, 查看它提供了哪些命令和选项是一个好习惯

先导键在配置插件时尤其有用, 因为它可以为插件提供完整的命名空间

模式

Vim 利用不同的模式来响应不同的用户输入, Vim 有 7 种主模式, 理解每一种模式的用途非常重要, 有助于更好的掌握 Vim 的方方面面

正常模式

打开 Vim 时默认为正常模式(normal mode), 在其他模式中按一次 Esc 键, 也会返回正常模式

命令模式/ex模式

命令模式(command-line mode) 的进入方式是输入 : 或者用 / 或 ? 进行搜索, 进入命令模式之后, 可以输入一条命令, 然后按 Enter 键执行命令

ex 模式是为了兼容 Vim 的前身 Ex, 进入方式是按 Q 键, ex 模式支持执行多个命令而无须退出

插入模式

插入模式(insert mode) 用于输入文本 (状态栏标识文本 --INSERT--)

可视模式/选择模式

可视模式(visual mode) 支持文本的任意选择, 如果需要选择的文本不属于已定义的文本对象, 则这个模式非常有用, 进入可视模式有以下几种方式:

  • v 进入字符可视模式 (状态栏标识文本 --VISUAL--)
  • V 进入行可视模式 (状态栏标识文本 --VISUAL LINE--)
  • Ctrl+v 组合键进入块可视模式 (状态栏标识文本 --VISUAL BLOCK--)

选择模式(select mode) 模拟的是其他编辑器中的选择模式, 输入任意可打印字符可立即删除选中文本, 然后进入插入模式, 这个模式只应用于特定和受限的场合

替换模式/虚拟替换模式

替换模式(replace mode) 输入的文本会覆盖已有的文本, 进入方式是按 R 键 (状态栏标识文本 --REPLACE--)

虚拟替换模式(virtual-replace mode) 和替换模式类似, 只不过操作的对象是屏幕上的显示, 而非直接针对文件中的字符

终端模式

终端模式(terminal mode) 出现在 Vim8.1 版本中, 此模式支持在一个分割窗口中运行一个终端环境, 进入终端模式的命令为 :terminal

文本

Vim 中有两种单词对象:

  • 狭义单词(word) 指由空白字符(比如空格, 制表符, 换行符)分隔的字母, 数字和下划线组成的序列
  • 广义单词(WORD) 指由空格分隔的任何非字符组成的序列

光标

在 Vim 中可以用方向键或字母 h(左) j(下) k(上) l(右) 来逐个字符的移动光标, 在命令之前可以加数字, 表示重复这条命令的次数

Vim 支持通过行号移动

编辑

在 Vim 中使用 i 键进入插入模式, 插入文本的位置为光标的位置, 除此之外, 还有其他进入插入模式的便捷方法

搜索

大多数情况下, Vim 中最快的文本浏览方式是搜索指定的字符串

Vim 支持跨文件的搜索

撤销

和其他编辑器一样, Vim 也记录了每一步操作

Vim 支持在不同会话之间持久保存撤销历史, 从而允许撤销几天前的操作

寄存器

Vim 支持多种寄存器, 每个寄存器用字母, 数字或特殊符号来标识, 寄存器的访问方式是引号键 " 接寄存器标识符, 然后是针对指定寄存器的操作

复制(y) 粘贴(p) 使用的是默认的无名寄存器, 用双引号 " 标识

a~z 标识的寄存器用于手动复制粘贴, 0~9 标识的寄存器存储的是最后 10 次删除操作的历史记录

一些很有用的只读寄存器:

  • % 存储当前文件名
  • # 存储上次打开的文件名
  • . 存储最后插入的文本
  • : 存储最后执行的命令

Vim 中有如下两种内置的寄存器用于和外部交互:

  • * 寄存器表示系统的主粘贴板

    macOS 和 Windows 系统中的默认粘贴板, 在 Linux 系统中为终端的鼠标选择内容

  • + 寄存器称为粘贴板选择器

    只针对 Linux 系统, 用于 Windows 风格的 Ctrl+c 和 Ctrl+v 组合键

交换文件

默认情况下, Vim 用交换文件跟踪文件的变化情况, 当用户编辑文件的时候, Vim 会在原始文件的目录下, 自动产生 .swp 后缀的交换文件, 交换文件的作用是恢复文件内容, 当用户的会话或系统崩溃, 再次用 Vim 打开同一个文件时, 就会看到交换文件的操作提示
