Schema

GraphQL-Java 提供了通过代码编程定义 schema 的方式, 详见 schema

/* 示例: 代码方式 */
GraphQLObjectType hero = GraphQLObjectType.newObject()
    .name("Hero")
    .field(f -> f.name("id").type(Scalars.GraphQLID))
    .field(f -> f.name("name").type(Scalars.GraphQLString))
    .build();
GraphQLObjectType query = GraphQLObjectType.newObject()
    .name("Query")
    .field(f -> f.name("hero").type(hero)
           .argument(a -> a.name("id").type(Scalars.GraphQLID)))
    .build();
GraphQLSchema schema = GraphQLSchema.newSchema()
    .query(query)
    .build();
"等同于以下示例(GraphQL SDL)"
schema {
    query: Query
}
type Query {
    hero(id: ID): Hero
}
type Hero {
    id: ID
    name: String
}

推荐使用 GraphQL SDL 语言, 可以将 schema 字符串写到代码变量里, 或者以文本形式写到 .graphql 为扩展名的文件, 再读取到应用内存中

类型解析器 TypeResolver 用于决定具体值的类型, 这是接口和联合需要的功能

/* 示例: TypeResolver */
TypeResolver resolver = env -> {
    Object javaObject = env.getObject();  // 获取对象
    if (javaObject instanceof Human) {    // 根据Java类型返回定义的对象类型
        return env.getSchema().getObjectType("Human");
    } else if (javaObject instanceof Droid) {
        return env.getSchema().getObjectType("Droid");
    } else {
        return null;
    }
};

数据解析器 DataFetcher 为定义的类型和字段提供数据

/* 示例: DataFetcher */
DataFetcher<HeroDTO> fetcher = env -> {
    String id = env.getArgument("id");  // 可通过env获取参数
    return bizImpl.getHeroById(id);     // 返回获取的数据
};

运行时线路 RuntimeWiring 和 GraphQLSchema 组合成一个可执行的 schema

/* 示例: RuntimeWiring */
RuntimeWiring runtime = RuntimeWiring.newRuntimeWiring()
    .type("Query", trw -> trw.dataFetcher("hero", fetcher))  // 配置字段获取数据的解析器
    .type("Character", trw -> trw.typeResolver(resolver))    // 配置接口或联合确定类型的解析器
    .build();

File file = loadSchema("schema.graphql");                    // 读取schema文件
TypeDefinitionRegistry registry = new SchemaParser().parse(file);
GraphQLSchema schema = new SchemaGenerator().makeExecutableSchema(registry, runtime);

GraphQL-Java 允许将 schema 拆分成多个文件, 并在使用前将多个文件合并到一个逻辑单元中, 详见 modular

/* 示例: 合并多文件 */
SchemaParser parser = new SchemaParser();
SchemaGenerator generator = new SchemaGenerator();
TypeDefinitionRegistry registry = new TypeDefinitionRegistry();

File file = loadSchema("schema1.graphql");
File file2 = loadSchema("schema2.graphql");
File file3 = loadSchema("schema3.graphql");
registry.merge(parser.parse(file));
registry.merge(parser.parse(file2));
registry.merge(parser.parse(file3));

GraphQLSchema schema = generator.makeExecutableSchema(registry, runtime);

执行

要对 schema 执行查询, 需要构建一个 GraphQL 类型的对象, 详见 execution

/* 示例: 执行查询 */
GraphQL graphql = GraphQL.newGraphQL(schema).build();  // 创建GraphQL对象
// 创建查询的选择集
ExecutionInput query = ExecutionInput.newExecutionInput().query(
    """
    {
        hero {
            id
            name
        }
    }
    """
).build();
ExecutionResult result = graphql.execute(query);       // 执行查询
String response = new ObjectMapper()                   // 结果转Json格式
    .writerWithDefaultPrettyPrinter()                  // 结构化输出
    .writeValueAsString(result.toSpecification());
/*
ExecutionResult 是包含查询数据和错误列表的类型
result.toSpecification() 返回符合GraphQL响应规范的集合
*/

异步

实际上, GraphQL-Java 引擎查询数据时使用完全异步的执行, 并使调用看起来是同步的, 详见 async

/* 示例: 以下两种方式等同 */
// 方式一
ExecutionResult result = graphql.execute(query);
// 方式二
CompletableFuture<ExecutionResult> promise = graphql.executeAsync(query);
ExecutionResult result = promise.join();
/* 示例: 异步回调的方式 */
CompletableFuture<ExecutionResult> promise = graphql.executeAsync(query);
promise.thenAccept(result -> {
    if (result.isDataPresent()) {         // 判断是否执行成功
        HeroDTO data = result.getData();  // 获取查询结果的数据集合
    } else {
        String error = result.getErrors().get(0).getMessage();  // 获取错误信息
    }
});
promise.join();

策略

GraphQL-Java 提供了许多不同的策略, 用于处理不同的请求操作, 详见 strategy

/* 示例: 配置执行策略 */
GraphQL graphql = GraphQL.newGraphQL(schema)
    .queryExecutionStrategy(new AsyncExecutionStrategy())
    .mutationExecutionStrategy(new AsyncSerialExecutionStrategy())
    .subscriptionExecutionStrategy(new SubscriptionExecutionStrategy())
    .build();
// 以上代码相当于默认设置, 并且在大多数情况下是合理的选择

缓存

GraphQL-Java 引擎执行查询之前, 必须对请求的选择集进行解析和验证, GraphQL 可以通过配置缓存, 来避免重复解析验证的过程, 详见 cache

<!-- 示例使用caffeine作为缓存方案 -->
<dependency>
    <groupId>com.github.ben-manes.caffeine</groupId>
    <artifactId>caffeine</artifactId>
    <version>${latest-version}</version>
</dependency>
/* 示例: 配置缓存 */
Cache<String, PreparsedDocumentEntry> cache = Caffeine.newBuilder()
    .maximumSize(10_000)                      // 缓存的最大条目数量
    .expireAfterWrite(Duration.ofMinutes(5L)) // 缓存条目的过期时间
    .build();
PreparsedDocumentProvider preparsedCache = (executionInput, computeFunction) -> {
    Function<String, PreparsedDocumentEntry> mapCompute = key -> computeFunction.apply(executionInput);
    return cache.get(executionInput.getQuery(), mapCompute);
};
GraphQL graphQL = GraphQL.newGraphQL(schema)
    .preparsedDocumentProvider(preparsedCache)
    .build();

注意这是缓存解析的文档, 并不缓存查询结果, 建议将字段参数作为变量传递, 以达到高效的缓存命中率

订阅

GraphQL-Java 使用响应流实现订阅功能, 当客户端订阅的数据变更时, 服务端将发送新的结果, 详见 subscription

<!-- 示例使用reactive-streams处理响应流 -->
<dependency>
    <groupId>org.reactivestreams</groupId>
    <artifactId>reactive-streams</artifactId>
    <version>${latest-version}</version>
</dependency>
/* 示例: DataFetcher */
DataFetcher<Publisher<StockDTO>> fetcher = env -> {
    String code = env.getArgument("code");
    return buildPublisherForStock(code);
};
/* 示例: 订阅请求 */
ExecutionInput query = ExecutionInput.newExecutionInput().query(
    """
    subscription StockSubscription {
        stock(code: "IBM") {
            code
            count
            price
            datetime
        }
    }
    """
).build();
ExecutionResult executionResult = graphQL.execute(query);
Publisher<ExecutionResult> stockPriceStream = executionResult.getData();
AtomicReference<Subscription> subscriptionRef = new AtomicReference<>();
stockPriceStream.subscribe(new Subscriber<>() {
    @Override
    public void onSubscribe(Subscription s) {
        subscriptionRef.set(s);
        s.request(1);
    }
    @Override
    public void onNext(ExecutionResult er) {
        // 处理数据
        processStock(er.getData());
        // 等待下一个响应
        subscriptionRef.get().request(1);
    }
    @Override
    public void onError(Throwable t) {
        log.info(t.getMessage());
    }
    @Override
    public void onComplete() {
        log.info("the subscription has completed.");
    }
});

GraphQL-Java 只产生结果流, 它不关心通过网络发送数据的套接字等, 这与 GraphQL 规范无关, 更多关于响应流代码的详细信息, 可见 reactive-streams

其它流行的响应式实现, 详见 RxJava Reactor

解析器

GraphQL 的数据通过 graphql.schema.DataFetcher 解析器来获取, 详见 fetcher

/* 示例: DataFetcher */
DataFetcher fetcher = env -> {
    // 获取父数据的结果, 可以检查它以了解如何获取当前字段的特定信息
    T source = env.getSource();
    // 获取参数的集合
    Map<String, Object> arguments = env.getArguments();
    // 上下文对象是请求第一次执行时设置, 并在请求的生命周期内保持不变
    T context = env.getContext();
    // 请求的调用路径, 可以向上导航查看导致当前字段执行的类型和字段
    ExecutionStepInfo stepInfo = env.getExecutionStepInfo();
    // 当前请求选择的字段集合, 这有助于提前查看客户端想要的字段信息
    DataFetchingFieldSelectionSet set = env.getSelectionSet();
    // 请求会生成唯一的执行id, 可以使用它来记录每个独立的请求
    ExecutionId eid = env.getExecutionId();

    return null;
};

GraphQL-Java 中字段默认的数据解析器是 graphql.schema.PropertyDataFetcher 类, 它使用标准的表达式来获取对象的字段值, 默认情况下, 对于选择集中的字段, 它会从 POJO 的属性或 Map 的映射键中寻找同名的字段

在 GraphQL SDL 中可以使用 @fetch 指令来修改字段的映射

"示例: @fetch"
type Hero {
    id: ID
    name: String
    description: String @fetch(from: "desc")
}

字段选择

了解请求的选择集有助于提高 DataFetcher 的效率, 选择集的路径匹配使用 GLOB 语法, 详见 fieldselection

"示例: 请求的选择集"
{
    user(id: "1") {
        id
        name
        info {
            phone
            email
        }
    }
}
/* 示例: DataFetcher */
DataFetcher<UserDTO> fetcher = env -> {
    String userId = env.getArgument("id");                      // 获取参数
    DataFetchingFieldSelectionSet set = env.getSelectionSet();  // 获取请求的选择集
    if (env.contains("info/*")) {                               // 当有子类字段时一并获取
        List<SelectedField> fields = set.getFields("info/*");
        Set<String> infoFields = fields.stream().map(SelectedField::getName).collect(Collectors.toSet());
        // 伪代码 SELECT user.*, info.${infoFields} FROM user JOIN info WHERE user.id = ${userId}
        return bizImpl.getUser(userId, infoFields);
    }
    return bizImpl.getUserById(userId);
}

字段可见性

默认情况下, schema 中的每个字段都是可用的, GraphQL-Java 提供了阻止字段可见的配置, 支持使用正则表达式的语法, 详见 fieldvisibility

/* 示例: GraphqlFieldVisibility */
GraphqlFieldVisibility visibility = BlockedFields.newBlock()
    .addPattern("User.password")  // 指定完整路径的字段
    .addPattern(".*\\.cert")      // 指定末级为cert的字段
    .build();
GraphQLSchema.newSchema()
    .query(query)
    .codeRegistry(GraphQLCodeRegistry.newCodeRegistry()
        .fieldVisibility(visibility)
        .build())
    .build();

优化

GraphQL-Java 提供了一些特性, 以提高数据获取的性能

批处理

GraphQL-Java 支持使用数据加载器 org.dataloader.DataLoader 高效的处理批量的数据获取操作, 它会根据唯一键自动合并同级的对象请求参数, 并缓存获取的结果, 当再次获取相同键的对象时直接从缓存返回结果, 详见 batching java-dataloader

/* 示例: 查询 */
String query =
    """
    {
        hero(id = "2001") {
            name
            friends {  // 嵌套的好友列表可能会重复, 且二维数组会导致乘积数量的获取次数
                name   // DataLoader缓存会过滤已请求的对象, 且自动合并同级的请求来减少获取次数
                friends {
                    name
                }
            }
        }
    }
    """;
/* 示例: DataFetcher */
DataFetcher heroDataFetcher = env -> {
    String id = env.getArgument("id");
    DataLoader<String, Character> dataLoader = env.getDataLoader("character");
    return dataLoader.load(id);
};
DataFetcher friendsDataFetcher = env -> {
    Character c = env.getSource();  // 使用父结果中的数据来获取参数
    List<String> friendIds = c.getFriendIds();
    DataLoader<String, Character> dataLoader = env.getDataLoader("character");
    return dataLoader.loadMany(friendIds);
};
RuntimeWiring runtime = RuntimeWiring.newRuntimeWiring()
    .type(TypeRuntimeWiring.newTypeWiring("Query")
          .dataFetcher("hero", heroDataFetcher()))
    .type(TypeRuntimeWiring.newTypeWiring("Hero")
          .dataFetcher("friends", friendsDataFetcher()))
    .build();
/* 示例: 批处理 */
GraphQL graphQL = GraphQL.newGraphQL(schema).build();
BatchLoader<String, Character> characterBatchLoader = keys -> CompletableFuture
    .supplyAsync(() -> bizImpl.getCharacterByIds(keys));
DataLoader<String, Character> characterDataLoader = DataLoader.newDataLoader(characterBatchLoader);
DataLoaderRegistry registry = new DataLoaderRegistry();
registry.register("character", characterDataLoader);  // 配置数据加载器
ExecutionInput input = ExecutionInput.newExecutionInput()
    .query(query)
    .dataLoaderRegistry(registry)
    .build();
ExecutionResult executionResult = graphQL.execute(input);  // 此示例最多只会触发3次数据获取操作

数据加载器 DataLoader 的实例范围很重要, 如果数据操作需要按用户角色返回特定数据的情况, 则需要为不同的用户角色分别实例化数据加载器, 以避免缓存数据提供给错误的用户

缓存

通过配置数据加载器的缓存实现, 改为使用 Memcached 或 Redis 等外部缓存, 则可以实现缓存数据的跨节点共享

<!-- 示例使用redis缓存 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
    <version>${latest-version}</version>
</dependency>
/* 示例: 配置redis缓存 */
CacheMap<String, Object> cacheMap = new CacheMap<>() {
    @Autowired  // 注入redis操作对象
    private RedisTemplate<String, Object> redisImpl;
    @Override
    public boolean containsKey(String key) {
        return redisImpl.hasKey(key);
    }
    @Override
    public Object get(String key) {
        return redisImpl.opsForValue().get(key);
    }
    @Override
    public CacheMap<String, Object> set(String key, Object value) {
        redisImpl.opsForValue().set(key, value);
        return this;
    }
    @Override
    public CacheMap<String, Object> delete(String key) {
        redisImpl.delete(key);
        return this;
    }
    @Override
    public CacheMap<String, Object> clear() {
        redisImpl.keys("*").forEach(redisImpl::delete);
        return this;
    }
};
DataLoaderOptions options = DataLoaderOptions
    .newOptions().setCacheMap(cacheMap);  // 配置指定的缓存实现
DataLoader<String, Character> dataLoader = DataLoader
    .newDataLoader(characterBatchLoader, options);

延迟

在执行查询时, 通常有两类数据, 需要立即获取的数据和可以稍后获取的数据, GraphQL-Java 使用 @defer 指令实现了对延迟获取的支持, 详见 defer

/* 示例: 延迟 */
ExecutionInput query = ExecutionInput.newExecutionInput().query(
    """
    {
        post {
            postText           // 直接获取的字段
            comments @defer {  // 被标记@defer的字段将通过扩展使用订阅来获取
                commentText
            }
            reviews @defer {
                reviewText
            }
        }
    }
    """
).build();
ExecutionResult result = graphql.execute(query);
Map map = result.getData();       // 立即获取的数据
// 通过扩展获取订阅
Publisher<ExecutionResult> deferred = (Publisher<ExecutionResult>)result.getExtensions().get(GraphQL.DEFERRED_RESULTS);
// 订阅延迟的数据
deferred.subscribe(new Subscriber<>() {
    @Override
    public void onNext(ExecutionResult er) {
        Map map2 = er.getData();  // 延迟获取的数据
    }
});

扩展

GraphQL-Java 允许应用声明非标准的自定义扩展, 注意这在跨语言的开发中, 可能会导致难处理的语义

标量

GraphQL 规定所有实现都必须具有 5 种基础的标量类型, GraphQL-Java 可以通过 Scalars 对象获取已定义的静态标量, 详见 scalars

/* 示例: 内置标量 */
Scalars.GraphQLString
Scalars.GraphQLBoolean
Scalars.GraphQLInt
Scalars.GraphQLFloat
Scalars.GraphQLID

应用可以编写自定义的标量实现, 或者使用 GraphQL-Java 的扩展标量, 详见 extended-scalars

/* 示例: 自定义标量 */
public class CustomScalars {
    public static final GraphQLScalarType EMAIL = GraphQLScalarType.newScalar()
        .name("Email")                                       // 定义schema中使用的标量关键字
        .description("A custom scalar that handles emails")  // 描述性文本
        .coercing(new EmailCoercing())                       // 标量实现对象
        .build();
}
public class EmailCoercing implements Coercing<String, String> {
    @Override  // 将Java对象序列化为标量的输出形式
    public String serialize(Object dataFetcherResult) throws CoercingSerializeException {
        return serializeEmail(dataFetcherResult);
    }
    @Override  // 将标量的输入对象转化为Java对象
    public String parseValue(Object input) throws CoercingParseValueException {
        return parseEmailFromVariable(input);
    }
    @Override  // 将AST字面量转化为Java对象
    public String parseLiteral(Object input) throws CoercingParseLiteralException {
        return parseEmailFromAstLiteral(input);
    }
}
// 配置
RuntimeWiring runtime = RuntimeWiring.newRuntimeWiring()
    .scalar(CustomScalars.EMAIL)
    .build();

指令

GraphQL 规范包含两个内置的指令, 并且允许应用编写自定义的指令, 详见 directive

"示例: 自定义指令"
directive @dateFormat(format: String = "yyyy-MM-dd HH:mm:ss") on FIELD_DEFINITION
type user {
    id: ID!
    name: String!
    register_date: String @dateFormat(format)
    birth_date: String @dateFormat(format: "yyyy-MM-dd")
}
/* 示例: 指令实现 */
public class DateFormatter implements SchemaDirectiveWiring {
    @Override
    public GraphQLFieldDefinition onField(SchemaDirectiveWiringEnvironment<GraphQLFieldDefinition> environment) {
        String format = (String)environment.getDirective().getArgument("format").getValue();
        GraphQLFieldDefinition field = environment.getElement();
        GraphQLFieldsContainer parentType = environment.getFieldsContainer();
        DataFetcher originalFetcher = environment.getCodeRegistry().getDataFetcher(parentType, field);
        DataFetcher dataFetcher = dataFetchingEnvironment -> {
            Object value = originalFetcher.get(dataFetchingEnvironment);
            if (value instanceof Date) {
                return DateFormatUtils.format((Date)value, format);
            }
            return value;
        };
        environment.getCodeRegistry().dataFetcher(parentType, field, dataFetcher);
        return field;
    }
}
// 配置
RuntimeWiring runtime = RuntimeWiring.newRuntimeWiring()
    .directive("dateFormat", new DateFormatter())
    .build();

异常

GraphQL-Java 引擎遇到特殊情况时, 将抛出运行时异常(RuntimeException), exceptions 页面列出了执行过程可能抛出的运行时异常

应用可以创建自定义的异常和异常处理程序, 详见 exception

/* 示例: 自定义异常 */
public class CustomRuntimeException extends RuntimeException implements GraphQLError {
    @Override
    public List<SourceLocation> getLocations() {
        return null;
    }
    @Override  // 异常类型
    public ErrorClassification getErrorType() {
        return ErrorType.DataFetchingException;
    }
    @Override  // 额外数据
    public Map<String, Object> getExtensions() {
        return ImmutableMap.of(
            "extra", "extra data",
            "desc", "A custom exception."
        );
    }
}
/* 示例: 自定义异常处理 */
DataFetcherExceptionHandler handler = new SimpleDataFetcherExceptionHandler() {
    @Override
    public DataFetcherExceptionHandlerResult onException(DataFetcherExceptionHandlerParameters handlerParameters) {
        Throwable exception = handlerParameters.getException();
        if(exception instanceof CustomRuntimeException) {
             log.info(exception.getMessage());
        }
        return super.onException(handlerParameters);
    }
};
// 配置
ExecutionStrategy strategy = new AsyncExecutionStrategy(handler);
GraphQL graphal = GraphQL.newGraphQL(schema)
    .queryExecutionStrategy(strategy)
    .build();

跟踪器

GraphQL-Java 提供了 graphql.execution.instrumentation.Instrumentation 接口, 用以观察查询的执行, 也可以改变运行时的行为, 它的主要用例是监视和记录日志, 但也可以用于许多不同目的, 详见 instrumentation

GraphQL-Java 内置 3 个跟踪器的实现:

  • DataLoaderDispatcherInstrumentation 跟踪数据加载器的执行分派和统计
  • TracingInstrumentation 是遵循 apollo-tracing 中定义的跟踪格式的跟踪器, 跟踪的详细信息将在结果的扩展部分返回
  • FieldValidationInstrumentation 是用于查询执行之前验证字段及其参数的跟踪器, 如果在此过程中验证错误, 则中止查询执行, 错误将在查询结果中返回
/* 示例: 配置跟踪器 */
List<Instrumentation> instrumentations = Lists.newArrayList();
instrumentations.add(new DataLoaderDispatcherInstrumentation());
instrumentations.add(new TracingInstrumentation());
// SimpleFieldValidation对象可使用addRule添加字段验证规则
instrumentations.add(new FieldValidationInstrumentation(new SimpleFieldValidation()));
GraphQL.newGraphQL(schema)
    // ChainedInstrumentation允许组合跟踪链
    .instrumentation(new ChainedInstrumentation(instrumentations))
    .build();
