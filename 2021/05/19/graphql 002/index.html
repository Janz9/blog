GraphQL 示例

发表于 2021-05-19 更新于 2021-08-15 分类于 开发 ， Java 阅读次数： 本文字数： 17k 阅读时长 ≈ 15 分钟

示例

GraphQL 支持的开发语言, 参考 language

Java 开发, 参考 graphql-java graphql-spring-boot

Schema

GraphQL-Java 提供了通过代码编程定义 schema 的方式, 详见 schema

推荐使用 GraphQL SDL 语言, 可以将 schema 字符串写到代码变量里, 或者以文本形式写到 .graphql 为扩展名的文件, 再读取到应用内存中

类型解析器 TypeResolver 用于决定具体值的类型, 这是接口和联合需要的功能

数据解析器 DataFetcher 为定义的类型和字段提供数据

运行时线路 RuntimeWiring 和 GraphQLSchema 组合成一个可执行的 schema

GraphQL-Java 允许将 schema 拆分成多个文件, 并在使用前将多个文件合并到一个逻辑单元中, 详见 modular

执行

要对 schema 执行查询, 需要构建一个 GraphQL 类型的对象, 详见 execution

异步

实际上, GraphQL-Java 引擎查询数据时使用完全异步的执行, 并使调用看起来是同步的, 详见 async

策略

GraphQL-Java 提供了许多不同的策略, 用于处理不同的请求操作, 详见 strategy

缓存

GraphQL-Java 引擎执行查询之前, 必须对请求的选择集进行解析和验证, GraphQL 可以通过配置缓存, 来避免重复解析验证的过程, 详见 cache

注意这是缓存解析的文档, 并不缓存查询结果, 建议将字段参数作为变量传递, 以达到高效的缓存命中率

订阅

GraphQL-Java 使用响应流实现订阅功能, 当客户端订阅的数据变更时, 服务端将发送新的结果, 详见 subscription

GraphQL-Java 只产生结果流, 它不关心通过网络发送数据的套接字等, 这与 GraphQL 规范无关, 更多关于响应流代码的详细信息, 可见 reactive-streams

其它流行的响应式实现, 详见 RxJava Reactor

解析器

GraphQL 的数据通过 graphql.schema.DataFetcher 解析器来获取, 详见 fetcher

GraphQL-Java 中字段默认的数据解析器是 graphql.schema.PropertyDataFetcher 类, 它使用标准的表达式来获取对象的字段值, 默认情况下, 对于选择集中的字段, 它会从 POJO 的属性或 Map 的映射键中寻找同名的字段

在 GraphQL SDL 中可以使用 @fetch 指令来修改字段的映射

字段选择

了解请求的选择集有助于提高 DataFetcher 的效率, 选择集的路径匹配使用 GLOB 语法, 详见 fieldselection

字段可见性

默认情况下, schema 中的每个字段都是可用的, GraphQL-Java 提供了阻止字段可见的配置, 支持使用正则表达式的语法, 详见 fieldvisibility

优化

GraphQL-Java 提供了一些特性, 以提高数据获取的性能

批处理

GraphQL-Java 支持使用数据加载器 org.dataloader.DataLoader 高效的处理批量的数据获取操作, 它会根据唯一键自动合并同级的对象请求参数, 并缓存获取的结果, 当再次获取相同键的对象时直接从缓存返回结果, 详见 batching java-dataloader

数据加载器 DataLoader 的实例范围很重要, 如果数据操作需要按用户角色返回特定数据的情况, 则需要为不同的用户角色分别实例化数据加载器, 以避免缓存数据提供给错误的用户

缓存

通过配置数据加载器的缓存实现, 改为使用 Memcached 或 Redis 等外部缓存, 则可以实现缓存数据的跨节点共享

延迟

在执行查询时, 通常有两类数据, 需要立即获取的数据和可以稍后获取的数据, GraphQL-Java 使用 @defer 指令实现了对延迟获取的支持, 详见 defer

扩展

GraphQL-Java 允许应用声明非标准的自定义扩展, 注意这在跨语言的开发中, 可能会导致难处理的语义

标量

GraphQL 规定所有实现都必须具有 5 种基础的标量类型, GraphQL-Java 可以通过 Scalars 对象获取已定义的静态标量, 详见 scalars

应用可以编写自定义的标量实现, 或者使用 GraphQL-Java 的扩展标量, 详见 extended-scalars

指令

GraphQL 规范包含两个内置的指令, 并且允许应用编写自定义的指令, 详见 directive

异常

GraphQL-Java 引擎遇到特殊情况时, 将抛出运行时异常(RuntimeException), exceptions 页面列出了执行过程可能抛出的运行时异常

应用可以创建自定义的异常和异常处理程序, 详见 exception

跟踪器

GraphQL-Java 提供了 graphql.execution.instrumentation.Instrumentation 接口, 用以观察查询的执行, 也可以改变运行时的行为, 它的主要用例是监视和记录日志, 但也可以用于许多不同目的, 详见 instrumentation

GraphQL-Java 内置 3 个跟踪器的实现:

  • DataLoaderDispatcherInstrumentation 跟踪数据加载器的执行分派和统计
  • TracingInstrumentation 是遵循 apollo-tracing 中定义的跟踪格式的跟踪器, 跟踪的详细信息将在结果的扩展部分返回
  • FieldValidationInstrumentation 是用于查询执行之前验证字段及其参数的跟踪器, 如果在此过程中验证错误, 则中止查询执行, 错误将在查询结果中返回
