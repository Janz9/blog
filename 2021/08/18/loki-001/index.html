Loki 入门

发表于 2021-08-18 更新于 2021-09-05 分类于 运维 阅读次数： 本文字数： 5.5k 阅读时长 ≈ 5 分钟

入门

Loki 是一个功能齐全的日志聚合系统的核心, 专为高效保存日志消息而优化, 与其它日志系统不同, Loki 不索引原始日志消息, 而是基于标签构建索引

Loki 的组件在内部称为模块 (module), 单体模式部署时, 每个组件都运行在同一进程中, 通过 Loki 配置的 gRPC 端口在本地网络中连接, 以进行组件之间的相互通信, Loki 主要由 5 个组件构成:

  • Distributor 负责处理客户端传入的流, 这是日志数据写入路径中的第一站, 分发器将验证每个流的正确性, 确保其在配置的租户 (或全局) 限制内, 然后将有效的数据块分批, 并行发送到匹配的 Ingester
  • Ingester 负责接收数据块, 当数据达到一定数量或期限时, 将数据块刷新到数据库中, 其对数据块和索引分别使用单独的数据库, 因为它们的数据类型不同
  • Querier 基于 LogQL 查询语言, 以检索存储库和 Ingester 中尚未刷新的数据的索引, 并返回匹配的日志查询结果
  • Query Frontend 负责提供查询的 API 端点, 传入的查询请求保存在内部的队列中, 以便 Querier 提取执行
  • Ruler 分为记录规则和告警规则, 用于预计算表达式和触发告警事件

安装

Loki 提供了多种安装方式, 详见 installation, 支持使用容器方式部署 grafana/loki

配置

Loki 通过 YAML 文件来配置, 详见 configuration

Loki 不附带任何内置的身份认证层, 如需对外暴露端口的方式部署, 需要在服务前置运行身份认证的反向代理, 例如使用 basic auth 或 OAuth2 代理的 Nginx 服务

启动

通过运行二进制文件 release 启动服务, 详见 local

Loki 提供了一组管理 API, 以促进自动化和集成, 详见 management-api

  • GET /ready 当服务准备好时, 该端点返回 200 状态码
  • POST /flush 该端点用于触发 ingester 组件刷新内存块到存储区, 主要用于本地测试
  • POST /ingester/flush_shutdown 该端点用于触发 ingester 组件刷新内存块到存储区并关闭
  • GET /config 该端点用于获取当前服务的配置
  • GET /loki/api/v1/status/buildinfo 该端点用于获取当前服务的构建信息, 包括版本, 修订, 分支, 构建日期, 构建用户和使用的 Go 版本

模式

Loki 的组件默认运行在同一进程, 称为单进程 (single process) 或单体模式 (monolithic mode), 详见 mode

Loki 设计支持扩展部署, 每个组件都可以分解为单独的流程, 并且配置允许组件的单独扩展, 称为横向扩展 (horizontally scalable) 或微服务模式 (microservices mode), 详见 scalability

在扩展 Loki 时, 应考虑按组件划分多个 Loki 实例, 而不是按 Loki 单体进程, libsonnet 提供了文件演示划分组件以及根据资源使用情况扩展的配置

存储

与其它日志记录系统不同, Loki 的构建理念是只为日志的标签生成索引, 日志数据本身被压缩并以块的形式存储在对象存储区中, 这显著降低了日志记录的成本, 详见 storage

Loki 支持的索引存储实现, 参考 index

Loki 支持的块存储实现, 参考 chunks

多租户

Loki 是一个多租户系统, 不同租户的数据是完全隔离的, 通过在端点的请求头 X-Scope-OrgID 传递租户 ID 来标识不同的租户, 详见 multi-tenancy

客户端

Loki 支持多种收集日志的客户端, 详见 clients, 第三方的客户端, 详见 unofficial

Promtail

promtail 是一个客户端代理, 负责将本地日志的内容附加标签, 并发送到指定的 Loki 实例, 通常将其部署到需要收集日志的应用所在的服务器上

DockerDriver

docker-driver 是官方提供的 Docker 插件, 该插件支持从 Docker 容器读取日志并发送到 Loki 实例

docker-for-windows 尚不支持 Docker 插件, 详情参考 docker-engine

Loki4j

Loki4j 是基于 Java 应用程序的 Logback 日志框架, 通过配置 Loki4jAppender 将应用日志发送到 Loki 的第三方工具

集成

Loki 原生支持大多数现代框架的技术集成

Prometheus

Loki 服务提供公开的 Prometheus 度量的指标端点 /metrics, 以方便集成到 Prometheus 系统的实时监控

Loki 时序指标列表, 参考 observability

Grafana

Grafana 提供完整的 Loki 可视化集成, 可以通过网页手动配置, 参考 datasources/loki
