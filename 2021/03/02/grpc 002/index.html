示例

gRPC 支持的语言及示例, 参考 languages

grpc-java

Java 开发, 参考 grpc-java examples

syntax = "proto3";  // 指定所使用的版本
import "google/protobuf/wrappers.proto";  // 导入其他文件中定义的消息结构

封装类型 google/protobuf 列表, 参考 protobuf

service

service Biz {  // 定义服务
  rpc insert(Entity) returns (Identification);                              // 一元模式
  rpc search(Query) returns (stream Entity);                                // 服务端流模式
  rpc delete(stream Identification) returns (google.protobuf.Int32Value);   // 客户端流模式
  rpc update(stream Entity) returns (stream Identification);                // 双向流模式
}

message

message Identification {  // 定义消息
  oneof identity {  // oneof表示同时最多只能设置一个字段
    int32 id = 1;
    string code = 2;
  }
}
message Query {
  repeated int32 id = 1;     // repeated表示可重复任意次数的字段
  optional string name = 2;  // optional表示可选参数
}
message Entity {
  int32 id = 1;
  string code = 2;
  string name = 3;
}
// 字段索引的取值范围为1~(2^29-1), 其中19000~19999用于预留编号

字段类型及其对应开发语言的类型, 见表 scalar, 解析消息时, 如果消息中特定类型的字段没有赋值, 则使用特定类型的默认值, 参考 default

option

option java_multiple_files = true;
option java_package = "com.demo";
option java_outer_classname = "BizProto";

可以使用多个选项注解 proto 文件中的个别声明, 选项不会更改声明的整体含义, 但可能会影响其在特定上下文中处理的方式, 选项可分为: 文件级, 消息级, 字段级

完整的选项列表定义在 google/protobuf/descriptor.proto 文件中, 第三方扩展的选项, 详见 extensions

protoc

定义的消息类型, 生成开发语言的源码文件时, 需要使用 protocol buffers 编译器 protoc

# 示例: 基于Java的编译命令
> protoc --proto_path=IMPORT_PATH --java_out=DST_DIR path/to/file.proto

不同开发语言的参考文档, 详见 reference protobuf java

插件 protoc-gen-grpc-java 为每个 .proto 文件生成数据类和一个 Grpc 类, 这个类包含以下组件:

  • ImplBase 映射实现到 grpc 服务的抽象类, 开发者需要继承并实现具体的服务逻辑
  • Stub 客户端实现

server

/* 示例: 服务端实现 */
@Log
public class BizImpl extends BizGrpc.BizImplBase {
    @Override
    public void insert(Entity request, StreamObserver<Identification> responseObserver) {
        // 参数校验
        if ("".equals(request.getName())) {
            responseObserver.onError(Status.INVALID_ARGUMENT.withDescription("name is not be blank.").asRuntimeException());
            return;
        }
        responseObserver.onNext(Identification.newBuilder().setId(request.getId()).build());
        responseObserver.onCompleted();
    }
    @Override
    public void search(Query request, StreamObserver<Entity> responseObserver) {
        responseObserver.onNext(Entity.newBuilder().setId(1).setName("name").build());
        responseObserver.onCompleted();
    }
    @Override
    public StreamObserver<Identification> delete(StreamObserver<Int32Value> responseObserver) {
        return new StreamObserver<Identification>() {
            int count = 0;
            boolean error = false;
            @Override
            public void onNext(Identification request) {
                if (request.hasId()) {
                } else if (request.hasCode()) {
                } else {
                    error = true;
                    responseObserver.onError(Status.UNIMPLEMENTED.withDescription("not supported").asRuntimeException());
                }
                count++;
            }
            @Override
            public void onError(Throwable t) {}
            @Override
            public void onCompleted() {
                if (!error) {
                    responseObserver.onNext(Int32Value.of(count));
                    responseObserver.onCompleted();
                }
            }
        };
    }
    @Override
    public StreamObserver<Entity> update(StreamObserver<Identification> responseObserver) {
        return new StreamObserver<Entity>() {
            @Override
            public void onNext(Entity request) {
                responseObserver.onNext(Identification.newBuilder().setId(request.getId()).build());
                responseObserver.onCompleted();
            }
            @Override
            public void onError(Throwable t) {}
            @Override
            public void onCompleted() {}
        };
    }

    /* 配置启动 */
    private static int port = 9090;
    @SneakyThrows
    public static void main(String[] args) {
        Server server = ServerBuilder.forPort(port)
            .addService(new BizImpl()).build()
            // 服务端始终会使用客户端所指定的压缩方法, 如果对应的压缩器没有注册, 则会向客户端返回一个Unimplemented状态
            // .compressorRegistry(CompressorRegistry.getDefaultInstance()) // 配置压缩
            .start();
        // 终端添加退出句柄
        Runtime.getRuntime().addShutdownHook(new Thread(() -> {
            if (Objects.nonNull(server)) {
                try {
                    // 有序关闭服务, 已存在的连接会等待30s完成调用, 新连接会被拒绝
                    server.shutdown().awaitTermination(30, TimeUnit.SECONDS);
                } catch (InterruptedException e) { e.printStackTrace(); }
                log.info("server shutdown.");
            }
        }));
        if (Objects.nonNull(server)) {
            log.info("server started, listening on " + port);
            server.awaitTermination(30, TimeUnit.SECONDS); // 终端等待30s
        }
    }
}

client

/* 示例: 客户端调用 */
@Log
public class BizImpl {
    private static BizGrpc.BizStub bizStub;           // 异步
    private static BizGrpc.BizBlockingStub bizStub2;  // 同步

    public void insert() {
        try {
            Identification identification = bizStub2.insert(Entity.newBuilder().setId(1).setName("").build());
            log.info("insert id: " + identification.getId());
        } catch (StatusRuntimeException e) {
            log.warning("error: " + e.getStatus().getDescription());
        }
    }

    public void search() {
        Iterator<Entity> entity = bizStub2.search(Query.newBuilder().setName("name").build());
        log.info("search: " + entity.next());
    }

    public void delete() {
        StreamObserver<Identification> response = bizStub.delete(new StreamObserver<Int32Value>() {
            @Override
            public void onNext(Int32Value value) {
                log.info("count: " + value.getValue());
            }
            @Override public void onError(Throwable t) {}
            @Override public void onCompleted() {}
        });
        response.onNext(Identification.newBuilder().setId(1).build());
        response.onNext(Identification.newBuilder().setCode("2").build());
        response.onCompleted();
    }

    public void update() {
        StreamObserver<Entity> response = bizStub.update(new StreamObserver<Identification>() {
            @Override
            public void onNext(Identification value) {
                log.info("id: " + value.getId());
            }
            @Override public void onError(Throwable t) {}
            @Override public void onCompleted() {}
        });
        response.onNext(Entity.newBuilder().setId(1).setName("name").build());
        response.onCompleted();
    }

    /* 配置启动 */
    private static String target = "localhost:9090";
    @SneakyThrows
    public static void main(String[] args) {
        ManagedChannel channel = ManagedChannelBuilder.forTarget(target)
            // .useTransportSecurity() // 使用TLS
            .usePlaintext() // 使用明文
            // .enableRetry() // 启用重试
            // .compressorRegistry(CompressorRegistry.getDefaultInstance()) // 配置压缩
            .build();
        bizStub = BizGrpc.newStub(channel)
            .withDeadline(Deadline.after(2L, TimeUnit.SECONDS)); // 配置客户端截止时间为2s
        bizStub2 = BizGrpc.newBlockingStub(channel);
        try {
            BizImpl biz = new BizImpl();
            biz.insert();
            biz.search();
            biz.delete();
            biz.update();
            Thread.sleep(500);
        } finally { channel.shutdown(); }
    }
}

安全

TLS

传输层安全协议(transport layer security, TLS) 旨在为两个应用程序之间的通信提供隐私性和数据完整性, 它可用于在 gRPC 服务端和客户端的应用程序之间提供安全连接, 示例见 tls

JWT

JWT 定义了在服务端和客户端传输身份信息的容器, 签名的 JWT 可用作自包含的访问令牌, 这意味着资源服务器无须与授权服务器通信来验证客户端的令牌, 它可以通过验证签名来校验令牌, gRPC 内置了对 JWT 的支持, 示例见 jwt

元数据

在某些场景中, 因为预期共享的信息可能与业务上下文并没有关联, 所以它们不应该作为参数的一部分, 在这样的场景中, 可以使用 gRPC 元数据, 示例见 header

拦截器

gRPC 提供简单的拦截器, 用来在应用程序中执行一些通用逻辑, 比如日志, 身份验证, 授权, 性能度量指标, 跟踪以及其它自定义需求, gRPC 拦截器可以分为两类:

  • 一元拦截器(unary interceptor)
  • 流拦截器(streaming interceptor)

gRPC 拦截器可以用在服务端, 也可以用在客户端

server

/* 示例: 自定义服务端拦截器 */
public class MyInterceptor implements ServerInterceptor {
    @Override
    public <ReqT, RespT> ServerCall.Listener<ReqT> interceptCall(ServerCall<ReqT, RespT> call, Metadata headers, ServerCallHandler<ReqT, RespT> next) {
        // return next.startCall(call, headers);
        call = new ForwardingServerCall.SimpleForwardingServerCall<ReqT, RespT>(call) {
            @Override  // 服务端接收请求时
            public void request(int numMessages) {
                super.request(numMessages);
            }
            @Override  // 服务端发送元数据时
            public void sendHeaders(Metadata headers) {
                // StreamObserver<RespT>.onNext
                super.sendHeaders(headers);
            }
            @Override  // 服务端发送消息时
            public void sendMessage(RespT message) {
                // StreamObserver<RespT>.onNext
                super.sendMessage(message);
            }
            @Override  // 服务端关闭时
            public void close(Status status, Metadata trailers) {
                // StreamObserver<RespT>.onError
                // StreamObserver<RespT>.onCompleted
                super.close(status, trailers);
            }
        };
        return new ForwardingServerCallListener.SimpleForwardingServerCallListener<ReqT>(next.startCall(call, headers)) {
            @Override  // 客户端准备好时
            public void onReady() {
                super.onReady();
            }
            @Override  // 客户端发送消息时
            public void onMessage(ReqT message) {
                // StreamObserver<ReqT>.onNext
                super.onMessage(message);
            }
            @Override  // 客户端半关时
            public void onHalfClose() {
                // StreamObserver<ReqT>.onCompleted
                super.onHalfClose();
            }
            @Override  // 客户端完成时
            public void onComplete() {
                super.onComplete();
            }
            @Override  // 客户端退出时(截止时间等触发)
            public void onCancel() {
                // StreamObserver<ReqT>.onError
                super.onCancel();
            }
        };
    }
}

/* 示例: 自定义客户端拦截器 */
public class MyInterceptor implements ClientInterceptor {
    @Override
    public <ReqT, RespT> ClientCall<ReqT, RespT> interceptCall(MethodDescriptor<ReqT, RespT> method, CallOptions callOptions, Channel next) {
        // return next.newCall(method, callOptions);
        return new ForwardingClientCall.SimpleForwardingClientCall<ReqT, RespT>(next.newCall(method, callOptions)) {
            @Override
            public void start(Listener<RespT> responseListener, Metadata headers) {
                super.start(
                    new ForwardingClientCallListener.SimpleForwardingClientCallListener<RespT>(responseListener) {
                        @Override  // 服务端准备好时
                        public void onReady() {
                            super.onReady();
                        }
                        @Override  // 服务端发送元数据时
                        public void onHeaders(Metadata headers) {
                            // 
                            super.onHeaders(headers);
                        }
                        @Override  // 服务端发送消息时
                        public void onMessage(RespT message) {
                            super.onMessage(message);
                        }
                        @Override  // 服务端关闭时
                        public void onClose(Status status, Metadata trailers) {
                            super.onClose(status, trailers);
                        }
                    }, headers);
            }
            @Override  // 客户端发起请求时
            public void request(int numMessages) {
                super.request(numMessages);
            }
            @Override  // 客户端发送消息时
            public void sendMessage(ReqT message) {
                super.sendMessage(message);
            }
            @Override  // 客户端半关时
            public void halfClose() {
                super.halfClose();
            }
            @Override  // 客户端退出时
            public void cancel(@Nullable String message, @Nullable Throwable cause) {
                super.cancel(message, cause);
            }
        };
    }
}

负载均衡

在开发生产级 gRPC 应用程序时, 通常需要确保该应用程序能够满足高可用性和高扩展性的需求, 因此在生产环境中, 始终需要多个 gRPC 服务端, 在这些服务之间的分发处理需要使用负载均衡器, gRPC 通常使用以下两种主要的负载均衡机制:

  • 负载均衡代理

    在代理负载均衡场景中, 客户端向负载均衡器代理发起 RPC, 负载均衡器代理将 RPC 分发给一台可用的 gRPC 服务端, 负载均衡器代理会跟踪每台服务端的负载, 并为服务端分配负载提供不同的负载均衡算法, 例如, 可以使用 Nginx, Envoy 等作为 gRPC 应用程序的负载均衡器代理

  • 客户端负载均衡

    客户端负载均衡不借助中间代理层, 而是在 gRPC 客户端层实现负载均衡的逻辑, 客户端要知道多台 gRPC 服务端, 并为每个 RPC 选择一台 gRPC 服务端, gRPC 有两个默认支持的负载均衡算法:

    • pick_fist 始终选择第一个地址, 如果失败则会尝试下一个地址
    • round_robin 按顺序使用地址

负载均衡逻辑可以完全作为客户端应用程序 (也被称为厚客户端) 的一部分来进行开发, 也可以实现为一个专用的服务端, 称为后备负载均衡器, 详情见 balancer

负载测试

使用常规的工具很难对 gRPC 应用程序进行负载测试和基准测量, 这是因为这些应用程序都或多或少是与特定协议 (如 HTTP) 绑定的, 对于 gRPC 来说, 需要定制的负载测试工具

ghz 是使用 Go 语言实现的命令行工具, 能够生成对服务端的虚拟 RPC 负载, 从而实现对 gRPC 服务端的负载测试, 也能用在自动化持续集成环境中, 实现性能回归测试, 示例见 example

度量指标

度量指标是一段时间内测量数据的数字形式表示, 在讨论度量指标时, 通常会收集两种类型的数据:

  • 系统级的指标 例如 CPU 使用情况, 内存使用情况等
  • 应用级的指标 例如入站请求率, 请求错误率等

Prometheus 是一个用于系统监控和警告的开源工具集

java-grpc-prometheus

日志

在 gRPC Java 应用程序中, 没有控制日志级别的环境变量, 可以通过提供 logging 配置文件来启用日志

# 文件: logging.properties
handlers=java.util.logging.ConsoleHandler
io.grpc.netty.level=FINE
java.util.logging.ConsoleHandler.level=FINE
java.util.logging.ConsoleHandler.formatter=java.util.logging.SimpleFormatter
# 示例: 项目启动时指定配置文件
> java -jar app.jar -Djava.util.logging.config.file=logging.properties

生态系统

gRPC 生态系统项目不是核心 gRPC 实现的一部分, 但在构建和运行真正的 gRPC 应用程序时, 它们可能会非常有用, 这些项目是围绕 gRPC 构建的, 用来克服使用 gRPC 构建生态系统时所遇到的问题或限制

grpc-gateway 网关插件能够让 protocol buffers 编译器读取 gRPC 服务定义, 并生成反向代理服务端, 该服务端能够将 RESTful JSON API 翻译为 gRPC, 这是专门为 Go 编写的, 为了同时支持从 gRPC 和 HTTP 客户端应用程序调用 gRPC 服务

grpc-httpjson-transcoding 当客户端应用程序不支持 gRPC, 并且需要通过基于 HTTP 的 JSON 方式提供对 gRPC 服务的访问时, 该插件可以将 HTTP JSON 调用转换成 RPC, 并将它们传递给 gRPC 服务

server-reflection 是在 gRPC 服务端定义的一个服务, 它能提供该服务端上可公开访问的 gRPC 服务的信息, 因此客户端不需要预编译服务定义就能与服务端进行通信

grpc-middleware 是基于 Go 语言的一组拦截器, 辅助器和工具的集合, 在构建基于 gRPC 的应用程序时, 它允许在服务端或客户端以拦截器链的形式应用多个拦截器

health-checking gRPC 定义了一个健康检查协议(Health Checking API), 它根据服务端是否响应非健康状态来确定服务端的状态, 当服务端还没有准备好处理 RPC 或者根本没有响应健康探针的请求时, 响应就会表明服务端处于非健康状态或者在规定时间窗口内没有收到响应, 这样消费者就能采取相应的行动

grpc-health-probe 是社区提供的健康探针工具, 用来检查服务端的健康状态, 这是一个通用的客户端, 能够与 gRPC 标准的健康检查服务通信, 可以以 CLI 工具的方式来使用
